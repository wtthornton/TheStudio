# Story 58.4 — API Endpoint Verification Helper

**Epic:** [Epic 58 — Playwright Test Infrastructure](../../epic-58-playwright-test-infrastructure.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** test author,
**I want** a helper that calls API endpoints via Playwright's request context (sharing auth with the browser session),
**So that** page tests can verify their backing APIs return correct status codes and valid JSON.

## Description

Build helpers in `tests/playwright/lib/api_helpers.py` that call API endpoints using Playwright's `page.request` or `APIRequestContext`, sharing authentication cookies/headers with the browser session. Assert HTTP status codes, validate JSON response structure, and report failures with endpoint context.

## Tasks

- [ ] Create `tests/playwright/lib/api_helpers.py`
- [ ] Implement `assert_api_endpoint(page, method, path, expected_status, json_keys=None)` — call endpoint, assert status, optionally check JSON keys
- [ ] Implement `assert_api_returns_data(page, path, list_key=None)` — for list endpoints, assert non-empty array or valid empty response
- [ ] Implement `assert_api_fields(page, path, required_fields)` — check all required fields present in JSON response
- [ ] Implement `assert_api_no_error(page, path)` — call endpoint, assert no 4xx/5xx, no "error"/"traceback" in body
- [ ] Support both GET and POST methods with optional JSON body
- [ ] Support authenticated (X-User-ID + Basic Auth) and unauthenticated endpoints
- [ ] Export from `tests/playwright/lib/__init__.py`
- [ ] Write tests using `/healthz` and `/admin/health` as validation targets

## Acceptance Criteria

- [ ] `assert_api_endpoint()` correctly asserts status codes for GET and POST
- [ ] `assert_api_fields()` validates required JSON fields are present
- [ ] Auth headers from browser context are reused for API calls
- [ ] Error messages include endpoint path, expected vs actual status, and response body excerpt
- [ ] Works with both authenticated and unauthenticated endpoints

## Definition of Done

- [ ] All helpers implemented, exported, docstrings complete
- [ ] Tests pass against running stack
