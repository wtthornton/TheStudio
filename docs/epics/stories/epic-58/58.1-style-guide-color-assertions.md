# Story 58.1 — Style Guide Color & Token Assertion Library

**Epic:** [Epic 58 — Playwright Test Infrastructure](../../epic-58-playwright-test-infrastructure.md)
**Points:** 5 | **Size:** M
**Status:** Proposed

---

## User Story

**As a** test author writing Playwright page tests,
**I want** reusable assertion helpers that validate computed CSS colors against the style guide token definitions,
**So that** every page can be checked for color compliance without duplicating color extraction logic.

## Description

Build assertion helpers in `tests/playwright/lib/style_assertions.py` that extract computed CSS colors from page elements via `page.evaluate()` / `getComputedStyle()` and validate them against the style guide token definitions (Sections 4–5). Covers status colors (success/warning/error/info/neutral), trust tier colors (EXECUTE/SUGGEST/OBSERVE), role colors (ADMIN/OPERATOR), interactive colors (primary/secondary/destructive/ghost), and focus ring colors — for both light and dark themes.

## Tasks

- [ ] Create `tests/playwright/lib/style_assertions.py` with color extraction via `page.evaluate()`
- [ ] Implement `assert_status_colors(page, selector)` — validates green/yellow/red/blue/gray badge backgrounds and text against Section 5.1
- [ ] Implement `assert_trust_tier_colors(page, selector)` — validates EXECUTE (purple), SUGGEST (blue), OBSERVE (gray) per Section 5.2
- [ ] Implement `assert_role_colors(page, selector)` — validates ADMIN (red), OPERATOR (yellow) per Section 5.3
- [ ] Implement `assert_button_colors(page, selector, variant)` — validates primary/secondary/destructive/ghost per Section 5.4
- [ ] Implement `assert_interactive_hover(page, selector)` — hover and verify color change
- [ ] Implement `assert_focus_ring_color(page, selector)` — Tab to element, verify `--color-focus-ring` (blue-600 light / blue-500 dark)
- [ ] Add dark theme toggle support: set `data-theme="dark"` and re-validate all colors
- [ ] Export from `tests/playwright/lib/__init__.py`
- [ ] Write unit tests in `tests/playwright/test_lib_style_assertions.py`

## Acceptance Criteria

- [ ] `assert_status_colors()` correctly validates all 5 status colors (success, warning, error, info, neutral) for both light and dark themes
- [ ] `assert_trust_tier_colors()` validates purple/blue/gray for EXECUTE/SUGGEST/OBSERVE
- [ ] `assert_button_colors()` validates bg + text + hover for all 4 button variants
- [ ] `assert_focus_ring_color()` verifies focus ring appears with correct color on keyboard focus
- [ ] Color values are parsed from computed styles (rgb/rgba), not class names
- [ ] Dark theme assertions work when `[data-theme="dark"]` is set
- [ ] All helpers have docstrings with usage examples

## Technical Notes

- Color comparison should use a tolerance (±5 per RGB channel) to handle anti-aliasing
- Use `page.evaluate()` with `getComputedStyle(element).backgroundColor` etc.
- Style guide token hex values defined in `docs/design/07-THESTUDIO-UI-UX-STYLE-GUIDE.md` Sections 4.1–4.2
- Light theme: `--color-status-success-bg: var(--primitive-green-100)` = `#dcfce7`
- Dark theme: `--color-status-success-bg: rgba(22, 163, 74, 0.2)`
