# Story 58.3 — Component Recipe Validators

**Epic:** [Epic 58 — Playwright Test Infrastructure](../../epic-58-playwright-test-infrastructure.md)
**Points:** 5 | **Size:** M
**Status:** Proposed

---

## User Story

**As a** test author,
**I want** validators for each UI component recipe (cards, tables, badges, buttons, forms, alerts, empty states, error states),
**So that** page tests can verify component structure and styling against Section 9 of the style guide.

## Description

Build validators in `tests/playwright/lib/component_validators.py` for each component recipe in style guide Section 9. Each validator checks DOM structure, CSS classes, ARIA attributes, and visual properties.

## Tasks

- [ ] Create `tests/playwright/lib/component_validators.py`
- [ ] `validate_card(page, selector)` — check bg (white/gray-900), border (gray-200/gray-700), radius (rounded-lg = 8px), padding (p-4 or p-6)
- [ ] `validate_table(page, selector)` — check header bg (gray-50), th `scope="col"`, row hover, dividers, numeric cols right-aligned + font-mono
- [ ] `validate_badge(page, selector)` — check px-2 py-0.5, rounded, text-xs, font-semibold, correct color pairing per status
- [ ] `validate_button(page, selector, variant)` — check rounded-md, px-3 py-2, text-sm font-medium, min 24×24px touch target, focus-visible ring
- [ ] `validate_form_input(page, selector)` — check associated `<label>`, border, focus ring, `aria-describedby` for help text
- [ ] `validate_empty_state(page, selector)` — check heading + description + CTA button structure per Section 9.11
- [ ] `validate_alert(page, selector)` — check `role="alert"`, correct color variant, content structure per Section 9.9
- [ ] `validate_modal(page, selector)` — check `role="dialog"`, `aria-modal="true"`, `aria-labelledby`, focus trap, Escape closes
- [ ] Export all from `tests/playwright/lib/__init__.py`
- [ ] Write tests validating each validator against known page elements

## Acceptance Criteria

- [ ] All 8 component validators implemented
- [ ] Each validator returns a structured result (pass/fail + details)
- [ ] Table validator checks `scope="col"` on all `<th>` elements
- [ ] Button validator checks minimum touch target (24×24px)
- [ ] Form input validator checks label association (for/id or wrapping)
- [ ] Empty state validator checks heading + description + CTA presence
- [ ] All validators work on both light and dark theme

## Definition of Done

- [ ] All validators implemented, exported, docstrings complete
- [ ] Tests pass against real page elements
