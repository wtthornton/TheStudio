# Story 58.6 — WCAG 2.2 AA Accessibility Checker

**Epic:** [Epic 58 — Playwright Test Infrastructure](../../epic-58-playwright-test-infrastructure.md)
**Points:** 5 | **Size:** M
**Status:** Proposed

---

## User Story

**As a** test author,
**I want** accessibility assertion helpers covering WCAG 2.2 AA requirements,
**So that** every page can be automatically checked for accessibility compliance.

## Description

Build accessibility assertion helpers in `tests/playwright/lib/accessibility_helpers.py` covering contrast ratios, focus indicators, ARIA roles, keyboard navigation, semantic HTML, touch targets, and screen reader support per style guide Section 11.

## Tasks

- [ ] Create `tests/playwright/lib/accessibility_helpers.py`
- [ ] `assert_focus_visible(page)` — Tab through all interactive elements, verify each shows 2px solid focus ring
- [ ] `assert_keyboard_navigation(page)` — Tab order is logical, all interactive elements reachable
- [ ] `assert_aria_landmarks(page)` — Check `<nav>`, `<main>`, `<header>` present with `aria-label` when duplicated
- [ ] `assert_aria_roles(page, selector, expected_role)` — Check specific ARIA roles on elements
- [ ] `assert_table_accessibility(page, selector)` — All `<th>` have `scope="col"` or `scope="row"`
- [ ] `assert_form_accessibility(page, selector)` — All `<input>` have associated `<label>`, `aria-describedby` for help text
- [ ] `assert_touch_targets(page)` — All buttons/links meet minimum 24×24px
- [ ] `assert_no_color_only_indicators(page)` — Status indicators pair color with text/icon/shape
- [ ] `run_axe_audit(page)` — Inject axe-core, run audit, return violations
- [ ] Export from `tests/playwright/lib/__init__.py`
- [ ] Write tests validating checker against Dashboard

## Acceptance Criteria

- [ ] Focus indicator check verifies 2px solid ring with correct color
- [ ] Keyboard navigation test covers Tab, Shift+Tab, Enter, Space, Escape
- [ ] ARIA landmark check validates nav, main, header, aside
- [ ] Table accessibility validates scope on every th
- [ ] Form accessibility validates label association on every input
- [ ] Touch target check validates minimum 24×24px on all interactive elements
- [ ] axe-core audit returns structured violations list with element selectors
- [ ] All checks produce clear, actionable error messages

## Definition of Done

- [ ] All helpers implemented, exported, docstrings complete
- [ ] Tests pass against Dashboard page
