# Story 67.1 — Page Intent & Semantic Content

**Epic:** 67 — Compliance Scorecard: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/compliance`
**Heading:** "Compliance Scorecard"

## User Story

As a platform operator, I want the Compliance Scorecard page to render with the correct heading, semantic structure, and content so that I can verify per-repo compliance status and individual check results are displayed correctly.

## Description

This story validates that the Compliance Scorecard page at `/admin/ui/compliance` loads successfully, displays the "Compliance Scorecard" heading, and contains the expected semantic content. The test navigates to the page, confirms the heading, and verifies that overall PASS/FAIL status, individual check results (coverage, security, test requirements), and repo selector or repo context are all present.

## Tasks

- [ ] Write Playwright test that navigates to `/admin/ui/compliance`
- [ ] Assert page contains an `<h1>` or primary heading with text "Compliance Scorecard"
- [ ] Assert overall PASS/FAIL status badge is visible
- [ ] Assert individual check list is present with check results (check, requirement, coverage, security, test)
- [ ] Assert repo selector or default repo context is shown
- [ ] Assert each check row shows a pass/fail indicator
- [ ] Verify page returns HTTP 200 and does not show error states

## Acceptance Criteria

- [ ] Test navigates to `/admin/ui/compliance` and page loads without errors
- [ ] Page heading "Compliance Scorecard" is visible and semantically correct (h1 or equivalent)
- [ ] Overall PASS/FAIL status is prominently displayed
- [ ] Individual compliance checks are listed with their results
- [ ] Repo selector dropdown or default repo context is visible
- [ ] Check rows include pass/fail indicators for each compliance criterion
- [ ] No console errors or uncaught exceptions during page load

## Definition of Done

- Playwright test file created and passing
- Test covers all acceptance criteria
- Test runs green in CI pipeline
