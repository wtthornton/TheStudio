# Story 67.6 — Visual Snapshot Baseline

**Epic:** 67 — Compliance Scorecard: Playwright Full-Stack Test Suite
**Points:** 2
**Page:** `/admin/ui/compliance`

## User Story

As a platform operator, I want visual snapshot baselines for the Compliance Scorecard page so that unintended visual regressions are caught automatically in CI.

## Description

This story captures Playwright visual snapshots of the Compliance Scorecard page in its default state and key interactive states. Snapshots serve as the baseline for future regression detection. Any future visual change to the page will require explicit baseline updates, preventing accidental UI drift.

## Tasks

- [ ] Capture full-page screenshot of `/admin/ui/compliance` in default/loaded state
- [ ] Capture screenshot of scorecard card showing overall PASS/FAIL badge
- [ ] Capture screenshot of individual check rows with pass/fail indicators
- [ ] Capture screenshot of repo selector in default state
- [ ] Store all snapshots in the designated visual baseline directory
- [ ] Configure snapshot comparison threshold (allow minor anti-aliasing differences)

## Acceptance Criteria

- [ ] Full-page snapshot captured and committed as baseline
- [ ] Scorecard card snapshot captured showing PASS/FAIL badge
- [ ] Check rows section snapshot captured with pass/fail indicators
- [ ] Repo selector snapshot captured in default state
- [ ] Snapshots are deterministic — repeated runs produce matching images
- [ ] Snapshot comparison threshold configured to avoid false positives from anti-aliasing
- [ ] All baselines stored in the project's visual snapshot directory

## Definition of Done

- Playwright visual snapshot test file created and passing
- Baseline images committed to repository
- Snapshot tests run green in CI pipeline
