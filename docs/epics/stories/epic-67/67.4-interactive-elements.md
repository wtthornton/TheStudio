# Story 67.4 — Interactive Elements

**Epic:** 67 — Compliance Scorecard: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/compliance`

## User Story

As a platform operator, I want interactive elements on the Compliance Scorecard page to function correctly so that I can select repos, trigger compliance evaluation, and expand individual check details.

## Description

This story validates all interactive elements on the Compliance Scorecard page. Tests exercise the repo selector dropdown, the evaluate button, expandable check rows, and any dynamic content loading. Each interaction is tested end-to-end: triggering the action, verifying state changes, and confirming visual feedback.

## Tasks

- [ ] Test repo selector dropdown — select a different repo and verify compliance data updates
- [ ] Test evaluate button — click to trigger compliance re-evaluation and verify result updates
- [ ] Test expandable check rows — click a check row to expand details if supported
- [ ] Verify dynamic content updates without full page reload (HTMX partial swaps)
- [ ] Test keyboard interaction for interactive elements (Enter/Space to activate buttons, arrow keys for dropdowns)
- [ ] Assert loading states appear during async operations (evaluation in progress)
- [ ] Test that PASS/FAIL status updates after re-evaluation completes

## Acceptance Criteria

- [ ] Repo selector changes the displayed compliance data when a different repo is selected
- [ ] Evaluate button triggers a re-evaluation and the scorecard updates with new results
- [ ] Check rows respond to click events and show expanded detail if applicable
- [ ] No JavaScript errors during any interaction
- [ ] Dynamic updates use partial rendering (no full page reloads for in-page actions)
- [ ] All interactive elements are reachable via keyboard
- [ ] Loading indicators appear during evaluation and resolve when complete

## Definition of Done

- Playwright test file created and passing
- All interactive elements tested with user-flow scenarios
- Test runs green in CI pipeline
