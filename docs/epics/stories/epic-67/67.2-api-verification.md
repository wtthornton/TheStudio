# Story 67.2 — API Endpoint Verification

**Epic:** 67 — Compliance Scorecard: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/compliance`

## User Story

As a platform operator, I want the Compliance Scorecard API endpoints to return valid responses so that the frontend can reliably display compliance status, evaluation results, and operational targets.

## Description

This story validates the backend API endpoints that power the Compliance Scorecard page. Each endpoint is tested independently for correct HTTP status codes, response structure, and meaningful content. The tests confirm that compliance status, evaluation, compliance partials, and operational targets endpoints all return well-formed data.

## Tasks

- [ ] Write Playwright/API test for `GET /admin/repos/{repo_id}/compliance` — assert HTTP 200 and response contains compliance status
- [ ] Write Playwright/API test for `POST /admin/repos/{repo_id}/compliance/evaluate` — assert HTTP 200 and response contains evaluation results
- [ ] Write Playwright/API test for `GET /admin/ui/partials/compliance` — assert HTTP 200 and response contains HTML partial
- [ ] Write Playwright/API test for `GET /admin/ui/partials/targets` — assert HTTP 200 and response contains operational targets
- [ ] Verify response content types (JSON for API endpoints, HTML for partials)
- [ ] Assert compliance response includes overall status and individual check results

## Acceptance Criteria

- [ ] `GET /admin/repos/{repo_id}/compliance` returns HTTP 200 with JSON compliance status payload
- [ ] `POST /admin/repos/{repo_id}/compliance/evaluate` returns HTTP 200 with JSON evaluation results
- [ ] `GET /admin/ui/partials/compliance` returns HTTP 200 with HTML content
- [ ] `GET /admin/ui/partials/targets` returns HTTP 200 with operational targets data
- [ ] Compliance response includes overall PASS/FAIL and individual check results
- [ ] No 4xx or 5xx errors from any endpoint under normal conditions

## Definition of Done

- Playwright/API test file created and passing
- All four endpoints tested with status code and response body assertions
- Test runs green in CI pipeline
