# Story 67.5 — Accessibility WCAG 2.2 AA

**Epic:** 67 — Compliance Scorecard: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/compliance`

## User Story

As a platform operator using assistive technology, I want the Compliance Scorecard page to meet WCAG 2.2 AA standards so that I can navigate and understand compliance status, individual checks, and repo selection using screen readers, keyboard navigation, and other assistive tools.

## Description

This story validates WCAG 2.2 AA compliance for the Compliance Scorecard page. Tests cover semantic HTML structure, ARIA attributes, keyboard navigation, color contrast, and form accessibility. Special attention is given to PASS/FAIL badges (which must convey meaning through text in addition to green/red color), check result indicators, and the repo selector label.

## Tasks

- [ ] Run axe-core accessibility scan on the page and assert zero violations at AA level
- [ ] Assert PASS/FAIL badges use text labels ("PASS"/"FAIL") in addition to color (not color-only meaning)
- [ ] Assert check result indicators are accessible (not conveyed by color alone)
- [ ] Assert repo selector dropdown has a programmatically associated `<label>` or `aria-label`
- [ ] Assert page has a logical heading hierarchy (h1 > h2 > h3, no skipped levels)
- [ ] Assert all interactive elements are keyboard-focusable with visible focus indicators
- [ ] Assert color contrast ratios meet WCAG AA minimums (4.5:1 for text, 3:1 for large text)
- [ ] Assert PASS (green) and FAIL (red) badge text maintains sufficient contrast against badge background

## Acceptance Criteria

- [ ] axe-core scan returns zero WCAG 2.2 AA violations
- [ ] PASS/FAIL badges include text labels, not color alone (green/red supplemented with "PASS"/"FAIL" text)
- [ ] Check result indicators include text or icon with accessible name
- [ ] Repo selector has a programmatically associated label
- [ ] Heading hierarchy is logical and sequential
- [ ] All interactive elements receive keyboard focus and show visible focus indicators
- [ ] Color contrast meets AA minimums across all text, badges, and indicators

## Definition of Done

- Playwright test file created and passing with axe-core integration
- All accessibility assertions verified
- Test runs green in CI pipeline
