# Story 64.2 — Expert Performance: API Endpoint Verification

**Epic:** [Epic 64 — Expert Performance Test Suite](../../epic-64-experts-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** test engineer validating the Expert Performance page,
**I want** automated tests confirming all backing API endpoints return valid responses,
**So that** API regressions are caught before they break the UI.

## Description

Validate that all API endpoints backing the Expert Performance page return 200 with valid JSON. This covers the experts list endpoint, the expert registry endpoint, and the experts list partial endpoint used by HTMX.

## Tasks

- [ ] Test `GET /admin/experts` returns 200 with valid JSON
- [ ] Test `GET /admin/experts/registry` returns 200 with valid JSON
- [ ] Test `GET /admin/ui/partials/experts` returns 200 with valid HTML partial
- [ ] Test experts list response contains expert records with tier fields
- [ ] Test registry response contains expert definitions with capabilities
- [ ] Test experts list supports filter query parameters (tier, repo)
- [ ] Test no 500 errors on any endpoint

## Acceptance Criteria

- [ ] `GET /admin/experts` returns 200 with JSON containing expert records
- [ ] `GET /admin/experts/registry` returns 200 with JSON containing registry data
- [ ] `GET /admin/ui/partials/experts` returns 200 with HTML content
- [ ] Expert records contain expected fields (name, tier, confidence, weight, drift, score)
- [ ] Registry records contain expert definitions and capabilities
- [ ] Filter parameters are accepted without errors
- [ ] No 500 errors on any tested endpoint

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
