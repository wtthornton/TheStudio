# Story 64.5 — Expert Performance: Accessibility WCAG 2.2 AA

**Epic:** [Epic 64 — Expert Performance Test Suite](../../epic-64-experts-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** operator who relies on assistive technology,
**I want** the Expert Performance page to meet WCAG 2.2 AA standards,
**So that** I can effectively review expert roster data regardless of ability.

## Description

Validate that `/admin/ui/experts` meets WCAG 2.2 AA accessibility standards. Tests cover tier badges using text + color per Section 5.2, filter controls having labels, table `scope="col"` attributes, keyboard navigation, focus indicators, and contrast ratios.

## Tasks

- [ ] Test tier badges use text + color (not color alone) per Section 5.2
- [ ] Test filter controls have associated labels (tier filter, repo filter)
- [ ] Test expert table has `scope="col"` on header cells
- [ ] Test filter controls are keyboard accessible (Tab, Enter, Space)
- [ ] Test visible focus indicators on all interactive elements
- [ ] Test color contrast ratios meet 4.5:1 minimum for text
- [ ] Test table rows are navigable via keyboard
- [ ] Test page has no accessibility violations via axe-core scan

## Acceptance Criteria

- [ ] Tier badges convey tier through text label, not color alone
- [ ] All filter controls have programmatically associated labels
- [ ] Expert table header cells have `scope="col"` attribute
- [ ] Filter controls fully keyboard accessible
- [ ] Focus indicators visible on all interactive elements with 2px+ outline
- [ ] Text contrast ratios meet 4.5:1 WCAG AA minimum
- [ ] Table rows navigable via keyboard (arrow keys or Tab)
- [ ] axe-core scan returns zero violations

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
