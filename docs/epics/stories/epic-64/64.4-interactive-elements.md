# Story 64.4 — Expert Performance: Interactive Elements

**Epic:** [Epic 64 — Expert Performance Test Suite](../../epic-64-experts-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator using the Expert Performance page,
**I want** all interactive elements to function correctly,
**So that** I can filter experts and navigate to detail views without encountering broken controls.

## Description

Validate that all interactive elements on `/admin/ui/experts` function correctly. This includes tier filter changing displayed experts, table rows linking to expert detail views, and possible sort controls on metric columns.

## Tasks

- [ ] Test tier filter changes displayed experts (shadow, probation, trusted, all)
- [ ] Test repo filter changes displayed experts (if applicable)
- [ ] Test table rows link to expert detail view (if applicable)
- [ ] Test sort controls on metric columns change row order (if applicable)
- [ ] Test HTMX partial refresh updates expert list without full page reload
- [ ] Test filter reset/clear returns to default view showing all experts
- [ ] Test no console errors during interactions
- [ ] Test interactive elements have appropriate cursor styles

## Acceptance Criteria

- [ ] Changing tier filter updates the expert list to show only matching experts
- [ ] Changing repo filter updates the expert list accordingly
- [ ] Table row clicks navigate to expert detail (if detail view exists)
- [ ] Sort controls reorder table rows by selected metric
- [ ] HTMX partial refresh works without full page reload
- [ ] No console errors during any interaction
- [ ] All interactive elements show pointer cursor on hover

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
