# Story 64.1 — Expert Performance: Page Intent & Semantic Content

**Epic:** [Epic 64 — Expert Performance Test Suite](../../epic-64-experts-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator viewing the Expert Performance page,
**I want** automated tests confirming the page delivers its intended purpose,
**So that** regressions in page content or structure are caught before deployment.

## Description

Validate that `/admin/ui/experts` delivers its documented purpose: showing the AI expert roster with trust tiers (shadow/probation/trusted), confidence scores, and drift signals for tuning expert selection. Tests verify the page loads with the correct heading, contains all required semantic content, and communicates its purpose through domain-specific vocabulary and data structures.

## Tasks

- [ ] Test page loads with 200 and correct heading "Expert Performance"
- [ ] Test trust tier labels are visible (shadow, probation, trusted)
- [ ] Test performance metric columns are present (confidence, weight, drift, score)
- [ ] Test expert filtering is available (by tier or repo)
- [ ] Test expert list table or empty state is shown
- [ ] Test no HTML entity artifacts (`&#`) in visible text
- [ ] Test no rendering artifacts (NaN, undefined, null)
- [ ] Test no console errors

## Acceptance Criteria

- [ ] Page returns 200 with heading containing "Expert Performance"
- [ ] Trust tier labels (shadow, probation, trusted) visible in table or filter
- [ ] Performance metric columns present in expert table header
- [ ] Expert filtering controls available (tier filter, repo filter)
- [ ] Expert list table shows data rows or empty state message
- [ ] Zero console errors during test execution
- [ ] No rendering artifacts in visible text

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
