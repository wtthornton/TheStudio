# Story 64.3 — Expert Performance: Style Guide Compliance

**Epic:** [Epic 64 — Expert Performance Test Suite](../../epic-64-experts-playwright-suite.md)
**Points:** 5 | **Size:** M
**Status:** Proposed

---

## User Story

**As a** design-conscious operator viewing the Expert Performance page,
**I want** the page to conform to the UI/UX style guide,
**So that** the expert roster is visually consistent with the rest of the admin interface.

## Description

Validate that `/admin/ui/experts` conforms to `docs/design/07-THESTUDIO-UI-UX-STYLE-GUIDE.md`. Tests check color tokens, typography, spacing, and component recipes for the data table (expert list), tier badges (shadow=gray, probation=yellow, trusted=green), filter controls, and metric columns.

## Tasks

- [ ] Test page background and surface colors match style guide tokens
- [ ] Test heading typography (font-family, font-size, font-weight) matches Section 4.2
- [ ] Test data table styling matches component recipe (borders, row spacing, header styling)
- [ ] Test tier badges use correct color tokens: shadow=gray, probation=yellow, trusted=green per Section 5.2
- [ ] Test filter control styling matches component recipe
- [ ] Test metric column values use correct typography (monospace for numbers)
- [ ] Test spacing between sections matches style guide spacing scale
- [ ] Test both light and dark theme compliance

## Acceptance Criteria

- [ ] Background and surface colors match style guide tokens (light and dark)
- [ ] Typography matches Section 4.2 specifications
- [ ] Data table follows component recipe with correct borders and spacing
- [ ] Tier badges use correct colors: shadow=gray, probation=yellow, trusted=green
- [ ] Filter controls follow component recipe
- [ ] Metric values use appropriate typography
- [ ] Spacing between sections follows the spacing scale
- [ ] Both light and dark themes pass all checks

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
