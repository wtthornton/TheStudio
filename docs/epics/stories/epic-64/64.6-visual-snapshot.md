# Story 64.6 — Expert Performance: Visual Snapshot Baseline

**Epic:** [Epic 64 — Expert Performance Test Suite](../../epic-64-experts-playwright-suite.md)
**Points:** 2 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** developer modifying the Expert Performance page,
**I want** a visual snapshot baseline committed to the repository,
**So that** unintended visual regressions are detected by comparing against the baseline.

## Description

Capture and commit visual snapshot baselines for `/admin/ui/experts` in both light and dark themes. Snapshots cover the full page, the expert list table, tier badges (shadow/probation/trusted), filter controls, and metric columns. These baselines are used for pixel-diff regression detection in CI.

## Tasks

- [ ] Capture full-page snapshot in light theme
- [ ] Capture full-page snapshot in dark theme
- [ ] Capture expert list table component snapshot
- [ ] Capture tier badges (shadow, probation, trusted) component snapshot
- [ ] Capture filter controls component snapshot
- [ ] Capture empty state snapshot (if applicable)
- [ ] Commit baseline images to `tests/playwright/snapshots/experts/`

## Acceptance Criteria

- [ ] Full-page snapshots captured for both light and dark themes
- [ ] Component-level snapshots captured for expert table, tier badges, and filter controls
- [ ] Snapshots committed to `tests/playwright/snapshots/experts/`
- [ ] Snapshot comparison test passes with 0% diff against baseline
- [ ] Empty state snapshot captured if the page supports an empty state

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
