# Story 61.1 — Workflow Console: Page Intent & Semantic Content

**Epic:** [Epic 61 — Workflow Console Test Suite](../../epic-61-workflows-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator viewing the Workflow Console page,
**I want** automated tests confirming the page delivers its intended purpose,
**So that** regressions in page content or structure are caught before deployment.

## Description

Validate that `/admin/ui/workflows` delivers its documented purpose: pipeline execution state, status filtering, workflow list with duration and stage. Tests verify the page loads with correct heading, contains all required semantic content, and communicates its purpose through domain-specific vocabulary and data structures.

## Tasks

- [ ] Test page loads with 200 and correct heading "Workflow Console"
- [ ] Test status filter (select or form) is present and functional
- [ ] Test status labels (running, completed, failed, stuck) are referenced in the UI
- [ ] Test workflow list table or empty state is shown
- [ ] Test workflow entries show pipeline stage and duration when data is present
- [ ] Test no HTML entity artifacts (`&#`) in visible text
- [ ] Test no rendering artifacts (NaN, undefined, null)
- [ ] Test no console errors

## Acceptance Criteria

- [ ] Page returns 200 with heading containing "Workflow Console"
- [ ] Status filter control is present and contains expected status options
- [ ] Workflow list table or empty state message is displayed
- [ ] Workflow entries include pipeline stage and duration fields
- [ ] Zero console errors during test execution
- [ ] No rendering artifacts in visible text

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
