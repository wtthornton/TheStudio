# Story 61.2 — Workflow Console: API Endpoint Verification

**Epic:** [Epic 61](../../epic-61-workflows-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** test engineer,
**I want** every API endpoint backing the Workflow Console page verified,
**So that** the page's data layer is validated independently of rendering.

## Description

Verify all API endpoints that power the Workflow Console page return correct HTTP status codes and valid JSON responses with expected fields.

## Tasks

- [ ] Test `GET /admin/workflows` returns 200 with JSON array of workflow objects containing status, stage, and duration fields
- [ ] Test `GET /admin/workflows?status=running` filters correctly and returns only running workflows
- [ ] Test `GET /admin/workflows?status=failed` filters correctly and returns only failed workflows
- [ ] Test `GET /admin/workflows/metrics` returns 200 with JSON containing workflow metric counts
- [ ] Test `GET /admin/ui/partials/workflows` returns 200 with HTML content (HTMX partial)
- [ ] Test all endpoints return proper Content-Type headers
- [ ] Test error responses do not contain stack traces or internal details

## Acceptance Criteria

- [ ] All endpoints return 200 (or appropriate status) with valid JSON
- [ ] Response bodies contain expected fields
- [ ] No 4xx/5xx errors or traceback content
- [ ] Auth headers correctly forwarded

## Definition of Done

- [ ] All API tests pass against running stack
