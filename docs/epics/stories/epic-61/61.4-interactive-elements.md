# Story 61.4 — Workflow Console: Interactive Elements

**Epic:** [Epic 61](../../epic-61-workflows-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator,
**I want** every button, form, filter, and interactive control on the Workflow Console page tested,
**So that** broken interactions are caught before deployment.

## Description

Test all interactive elements on `/admin/ui/workflows`: status filter dropdown changes displayed workflows, table rows link to workflow detail, pagination if applicable. Verify clicks produce expected responses, HTMX swaps update the right DOM targets, and filters/selects modify displayed data.

## Tasks

- [ ] Test status filter dropdown is present and contains expected options (running, completed, failed, stuck)
- [ ] Test selecting a status filter value updates the displayed workflow list
- [ ] Test filter change triggers HTMX swap (not full page reload)
- [ ] Test workflow table rows are clickable and navigate to workflow detail page
- [ ] Test pagination controls work if workflow list exceeds page size
- [ ] Test HTMX-driven updates swap correct target elements
- [ ] Test no console errors during any interaction

## Acceptance Criteria

- [ ] All buttons produce visible feedback on click
- [ ] Status filter dropdown changes displayed workflows correctly
- [ ] HTMX-driven updates swap correct target elements
- [ ] No console errors during interactions

## Definition of Done

- [ ] All interaction tests pass
- [ ] Uses shared helpers from Epic 58.5
