# Story 68.1 — Page Intent & Semantic Content

**Epic:** 68 — Quarantine: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Quarantine page to clearly communicate its purpose through semantic HTML and meaningful content so that I can immediately understand I am viewing quarantined events and their failure reasons.

## Description

Verify that the `/admin/ui/quarantine` page renders with correct semantic structure, conveys the "Quarantine" heading, displays the event list table or an appropriate empty state, and uses the expected status vocabulary (pending, replayed, corrected, reason, quarantine, status). Failure reason filtering must be available via a select or text filter control.

## Tasks

- [ ] Write Playwright test navigating to `/admin/ui/quarantine`
- [ ] Assert page contains an `<h1>` or prominent heading with "Quarantine"
- [ ] Assert semantic landmark structure (`<main>`, `<nav>`, `<header>` as applicable)
- [ ] Assert event list renders as a `<table>` with appropriate column headers
- [ ] Assert empty state displays "No quarantined events" when no data present
- [ ] Assert status vocabulary tokens appear in rendered content (pending, replayed, corrected, reason, quarantine, status)
- [ ] Assert failure reason filter control is present (select or text input)
- [ ] Validate no broken heading hierarchy (h1 > h2 > h3 in order)

## Acceptance Criteria

- [ ] Playwright test navigates to the quarantine page and asserts heading text
- [ ] Semantic landmarks are present and correct
- [ ] Table structure renders with column headers or empty state is shown
- [ ] Status vocabulary tokens are present in the page content
- [ ] Failure reason filter is present and labeled
- [ ] Heading hierarchy has no skipped levels

## Definition of Done

- [ ] Playwright test written and passing
- [ ] Test committed to `tests/playwright/quarantine/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-68 tracker
