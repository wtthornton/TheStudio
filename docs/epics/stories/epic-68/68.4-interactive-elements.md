# Story 68.4 — Interactive Elements

**Epic:** 68 — Quarantine: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Quarantine page interactive elements to function correctly so that I can filter events by failure reason, replay quarantined events, and delete events with confirmation.

## Description

Verify that all interactive elements on `/admin/ui/quarantine` work as expected: the failure reason filter changes displayed events, the Replay button triggers a replay and shows the result, the Delete button requires confirmation before acting, and HTMX partial updates refresh content without full page reload.

## Tasks

- [ ] Write Playwright test for reason filter — select/type a reason and assert table content changes
- [ ] Write test for Replay button — click Replay on an event and assert success feedback
- [ ] Write test for Delete button — click Delete and assert confirmation dialog/prompt appears
- [ ] Write test for Delete confirmation — confirm deletion and assert event is removed from list
- [ ] Write test for Delete cancellation — cancel deletion and assert event remains
- [ ] Assert HTMX partial updates occur (no full page navigation on filter/action)
- [ ] Assert Replay result is reflected in status badge (e.g., status changes to "replayed")

## Acceptance Criteria

- [ ] Reason filter changes the displayed event list
- [ ] Replay button triggers replay and updates the event status
- [ ] Delete button shows confirmation before deleting
- [ ] Confirmed deletion removes the event from the list
- [ ] Cancelled deletion leaves the event in place
- [ ] Interactions use HTMX partial updates (no full page reload)

## Definition of Done

- [ ] Playwright interaction tests written and passing
- [ ] Test committed to `tests/playwright/quarantine/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-68 tracker
