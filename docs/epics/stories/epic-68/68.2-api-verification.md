# Story 68.2 — API Endpoint Verification

**Epic:** 68 — Quarantine: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want all quarantine API endpoints to return correct HTTP responses so that the UI can reliably list, inspect, replay, and delete quarantined events.

## Description

Verify each quarantine API endpoint responds with the expected status codes and content types. The list partial, detail partial, replay action, and delete action must all behave correctly under normal and error conditions.

## APIs Under Test

| Method | Endpoint | Expected |
|--------|----------|----------|
| GET | `/admin/ui/partials/quarantine` | 200, HTML partial |
| GET | `/admin/ui/partials/quarantine/{id}` | 200 (valid id) / 404 (invalid id) |
| POST | `/admin/ui/partials/quarantine/{id}/replay` | 200 or 302 on success |
| DELETE | `/admin/ui/partials/quarantine/{id}` | 200 or 204 on success |

## Tasks

- [ ] Write Playwright test (or API-level test) for `GET /admin/ui/partials/quarantine` — assert 200 and HTML content
- [ ] Write test for `GET /admin/ui/partials/quarantine/{id}` with valid ID — assert 200
- [ ] Write test for `GET /admin/ui/partials/quarantine/{id}` with invalid ID — assert 404 or error partial
- [ ] Write test for `POST /admin/ui/partials/quarantine/{id}/replay` — assert success response
- [ ] Write test for `DELETE /admin/ui/partials/quarantine/{id}` — assert success response
- [ ] Assert response content-types are `text/html` for partials

## Acceptance Criteria

- [ ] All four endpoints return expected status codes for valid requests
- [ ] Invalid ID returns 404 or appropriate error
- [ ] Replay action returns success indicator
- [ ] Delete action returns success indicator
- [ ] All responses are HTML partials (content-type text/html)

## Definition of Done

- [ ] Playwright / API tests written and passing
- [ ] Test committed to `tests/playwright/quarantine/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-68 tracker
