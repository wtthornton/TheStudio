# Story 63.2 — Metrics: API Endpoint Verification

**Epic:** [Epic 63 — Metrics Test Suite](../../epic-63-metrics-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** test engineer validating the Metrics page,
**I want** automated tests confirming all backing API endpoints return valid responses,
**So that** API regressions are caught before they break the UI.

## Description

Validate that all API endpoints backing the Metrics page return 200 with valid JSON. This covers single-pass success metrics, loopback metrics, reopen metrics, success gate status, and the metrics partial endpoint used by HTMX.

## Tasks

- [ ] Test `GET /admin/metrics/single-pass` returns 200 with valid JSON
- [ ] Test `GET /admin/metrics/loopbacks` returns 200 with valid JSON
- [ ] Test `GET /admin/metrics/reopen` returns 200 with valid JSON
- [ ] Test `GET /admin/metrics/success-gate` returns 200 with valid JSON
- [ ] Test `GET /admin/ui/partials/metrics` returns 200 with valid HTML partial
- [ ] Test single-pass response contains success rate fields
- [ ] Test loopbacks response contains breakdown categories
- [ ] Test reopen response contains attribution fields
- [ ] Test success-gate response contains gate status field
- [ ] Test no 500 errors on any endpoint

## Acceptance Criteria

- [ ] `GET /admin/metrics/single-pass` returns 200 with JSON containing success rate data
- [ ] `GET /admin/metrics/loopbacks` returns 200 with JSON containing loopback breakdown
- [ ] `GET /admin/metrics/reopen` returns 200 with JSON containing reopen attribution
- [ ] `GET /admin/metrics/success-gate` returns 200 with JSON containing gate status
- [ ] `GET /admin/ui/partials/metrics` returns 200 with HTML content
- [ ] All response payloads contain expected fields
- [ ] No 500 errors on any tested endpoint

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
