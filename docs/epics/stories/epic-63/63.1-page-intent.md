# Story 63.1 — Metrics: Page Intent & Semantic Content

**Epic:** [Epic 63 — Metrics Test Suite](../../epic-63-metrics-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator viewing the Metrics page,
**I want** automated tests confirming the page delivers its intended purpose,
**So that** regressions in page content or structure are caught before deployment.

## Description

Validate that `/admin/ui/metrics` delivers its documented purpose: a quality metrics dashboard showing success rates, gate status (PASSING/FAILING), loopback breakdown, and reopen attribution with time window context. Tests verify the page loads with the correct heading, contains all required semantic content, and communicates its purpose through domain-specific vocabulary and data structures.

## Tasks

- [ ] Test page loads with 200 and correct heading "Metrics"
- [ ] Test success rate shown with percentage (%)
- [ ] Test gate status shown (PASSING/FAILING/Insufficient)
- [ ] Test loopback breakdown visible (loopback, loop, retry categories)
- [ ] Test reopen rate with attribution visible
- [ ] Test time window context present (7d, 30d)
- [ ] Test no HTML entity artifacts (`&#`) in visible text
- [ ] Test no rendering artifacts (NaN, undefined, null)
- [ ] Test no console errors

## Acceptance Criteria

- [ ] Page returns 200 with heading containing "Metrics"
- [ ] Success rate displayed with percentage value
- [ ] Gate status displayed as PASSING, FAILING, or Insufficient
- [ ] Loopback breakdown section visible with loopback, loop, and retry categories
- [ ] Reopen rate with attribution data visible
- [ ] Time window selector present with 7d and 30d options
- [ ] Zero console errors during test execution
- [ ] No rendering artifacts in visible text

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
