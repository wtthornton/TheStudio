# Story 63.3 — Metrics: Style Guide Compliance

**Epic:** [Epic 63 — Metrics Test Suite](../../epic-63-metrics-playwright-suite.md)
**Points:** 5 | **Size:** M
**Status:** Proposed

---

## User Story

**As a** design-conscious operator viewing the Metrics page,
**I want** the page to conform to the UI/UX style guide,
**So that** the metrics dashboard is visually consistent with the rest of the admin interface.

## Description

Validate that `/admin/ui/metrics` conforms to `docs/design/07-THESTUDIO-UI-UX-STYLE-GUIDE.md`. Tests check color tokens, typography, spacing, and component recipes for KPI cards (success rate, gate status), breakdown tables, time window selector, and status badges (PASSING/FAILING).

## Tasks

- [ ] Test page background and surface colors match style guide tokens
- [ ] Test heading typography (font-family, font-size, font-weight) matches Section 4.2
- [ ] Test KPI card styling matches component recipe (borders, padding, shadow)
- [ ] Test status badges (PASSING/FAILING) use correct color tokens per Section 5.1
- [ ] Test breakdown table styling matches table component recipe
- [ ] Test time window selector styling matches button/select component recipe
- [ ] Test percentage values use correct typography weight
- [ ] Test spacing between sections matches style guide spacing scale
- [ ] Test both light and dark theme compliance

## Acceptance Criteria

- [ ] Background and surface colors match style guide tokens (light and dark)
- [ ] Typography matches Section 4.2 specifications
- [ ] KPI cards follow component recipe with correct borders, padding, and shadow
- [ ] PASSING badge uses success color token, FAILING badge uses error color token
- [ ] Breakdown tables follow table component recipe
- [ ] Time window selector follows component recipe
- [ ] Spacing between sections follows the spacing scale
- [ ] Both light and dark themes pass all checks

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
