# Story 63.5 — Metrics: Accessibility WCAG 2.2 AA

**Epic:** [Epic 63 — Metrics Test Suite](../../epic-63-metrics-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** operator who relies on assistive technology,
**I want** the Metrics page to meet WCAG 2.2 AA standards,
**So that** I can effectively access quality metrics regardless of ability.

## Description

Validate that `/admin/ui/metrics` meets WCAG 2.2 AA accessibility standards. Tests cover KPI number ARIA labels, gate status badges communicating state through text (not just color), percentage values being screen-reader friendly, keyboard navigation, focus indicators, and contrast ratios.

## Tasks

- [ ] Test KPI numbers have appropriate ARIA labels describing the metric
- [ ] Test gate status badges have text not just color (PASSING/FAILING visible as text)
- [ ] Test percentage values are screen-reader friendly (e.g., "85 percent" not just "85%")
- [ ] Test time window selector is keyboard accessible (Tab, Enter, Space)
- [ ] Test visible focus indicators on all interactive elements
- [ ] Test color contrast ratios meet 4.5:1 minimum for text
- [ ] Test breakdown tables have `scope="col"` on header cells
- [ ] Test page has no accessibility violations via axe-core scan

## Acceptance Criteria

- [ ] KPI numbers have ARIA labels or accessible text describing the metric name
- [ ] Gate status badges convey state through text, not color alone
- [ ] Percentage values are accessible to screen readers
- [ ] Time window selector fully keyboard accessible
- [ ] Focus indicators visible on all interactive elements with 2px+ outline
- [ ] Text contrast ratios meet 4.5:1 WCAG AA minimum
- [ ] Breakdown table header cells have `scope="col"` attribute
- [ ] axe-core scan returns zero violations

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
