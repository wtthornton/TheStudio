# Story 63.6 — Metrics: Visual Snapshot Baseline

**Epic:** [Epic 63 — Metrics Test Suite](../../epic-63-metrics-playwright-suite.md)
**Points:** 2 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** developer modifying the Metrics page,
**I want** a visual snapshot baseline committed to the repository,
**So that** unintended visual regressions are detected by comparing against the baseline.

## Description

Capture and commit visual snapshot baselines for `/admin/ui/metrics` in both light and dark themes. Snapshots cover the full page, KPI cards, breakdown tables, status badges, and time window selector. These baselines are used for pixel-diff regression detection in CI.

## Tasks

- [ ] Capture full-page snapshot in light theme
- [ ] Capture full-page snapshot in dark theme
- [ ] Capture KPI cards component snapshot
- [ ] Capture breakdown table component snapshot
- [ ] Capture status badges (PASSING/FAILING) component snapshot
- [ ] Capture time window selector component snapshot
- [ ] Commit baseline images to `tests/playwright/snapshots/metrics/`

## Acceptance Criteria

- [ ] Full-page snapshots captured for both light and dark themes
- [ ] Component-level snapshots captured for KPI cards, breakdown tables, status badges, and time window selector
- [ ] Snapshots committed to `tests/playwright/snapshots/metrics/`
- [ ] Snapshot comparison test passes with 0% diff against baseline
- [ ] Both time window states (7d, 30d) have baselines captured

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
