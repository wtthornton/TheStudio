# Story 65.6 — Visual Snapshot Baseline

**Epic:** 65 — Tool Hub: Playwright Full-Stack Test Suite
**Points:** 2
**Page:** `/admin/ui/tools`

## User Story

As a platform operator, I want visual snapshot baselines for the Tool Hub page so that unintended visual regressions are caught automatically in CI.

## Description

This story captures Playwright visual snapshots of the Tool Hub page in its default state and key interactive states. Snapshots serve as the baseline for future regression detection. Any future visual change to the page will require explicit baseline updates, preventing accidental UI drift.

## Tasks

- [ ] Capture full-page screenshot of `/admin/ui/tools` in default/loaded state
- [ ] Capture screenshot of tool catalog section showing tool suite entries with approval badges
- [ ] Capture screenshot of access check form area (empty state)
- [ ] Capture screenshot of an expanded tool suite detail (if expandable)
- [ ] Store all snapshots in the designated visual baseline directory
- [ ] Configure snapshot comparison threshold (allow minor anti-aliasing differences)

## Acceptance Criteria

- [ ] Full-page snapshot captured and committed as baseline
- [ ] Tool catalog section snapshot captured showing approval badges
- [ ] Access check form area snapshot captured
- [ ] Snapshots are deterministic — repeated runs produce matching images
- [ ] Snapshot comparison threshold configured to avoid false positives from anti-aliasing
- [ ] All baselines stored in the project's visual snapshot directory

## Definition of Done

- Playwright visual snapshot test file created and passing
- Baseline images committed to repository
- Snapshot tests run green in CI pipeline
