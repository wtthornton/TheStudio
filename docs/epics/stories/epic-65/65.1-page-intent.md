# Story 65.1 — Page Intent & Semantic Content

**Epic:** 65 — Tool Hub: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/tools`
**Heading:** "Tool Hub"

## User Story

As a platform operator, I want the Tool Hub page to render with the correct heading, semantic structure, and content so that I can verify the tool catalog is displaying known tool suites with their approval statuses.

## Description

This story validates that the Tool Hub page at `/admin/ui/tools` loads successfully, displays the "Tool Hub" heading, and contains the expected semantic content. The test navigates to the page, confirms the page title and heading, and verifies that the tool suite catalog lists known suites (ruff, pytest, security) with their approval status indicators (observe/suggest/execute).

## Tasks

- [ ] Write Playwright test that navigates to `/admin/ui/tools`
- [ ] Assert page contains an `<h1>` or primary heading with text "Tool Hub"
- [ ] Assert the tool catalog section is present and contains tool suite entries
- [ ] Assert known tool suites appear in the catalog (ruff, pytest, security)
- [ ] Assert approval status is shown for each tool suite (observe, suggest, or execute)
- [ ] Assert access profiles or access check capability is visible on the page
- [ ] Verify page returns HTTP 200 and does not show error states

## Acceptance Criteria

- [ ] Test navigates to `/admin/ui/tools` and page loads without errors
- [ ] Page heading "Tool Hub" is visible and semantically correct (h1 or equivalent)
- [ ] Tool suite catalog lists known suites: ruff, pytest, security
- [ ] Each tool suite displays an approval status badge (observe/suggest/execute)
- [ ] Access profile information or access check form is present on the page
- [ ] No console errors or uncaught exceptions during page load

## Definition of Done

- Playwright test file created and passing
- Test covers all acceptance criteria
- Test runs green in CI pipeline
