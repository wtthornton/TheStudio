# Story 65.5 — Accessibility WCAG 2.2 AA

**Epic:** 65 — Tool Hub: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/tools`

## User Story

As a platform operator using assistive technology, I want the Tool Hub page to meet WCAG 2.2 AA standards so that I can navigate and understand the tool catalog using screen readers, keyboard navigation, and other assistive tools.

## Description

This story validates WCAG 2.2 AA compliance for the Tool Hub page. Tests cover semantic HTML structure, ARIA attributes, keyboard navigation, color contrast, and form accessibility. Special attention is given to approval status badges, which must convey meaning through text in addition to color.

## Tasks

- [ ] Run axe-core accessibility scan on the page and assert zero violations at AA level
- [ ] Assert all approval status badges use text labels in addition to color (not color-only meaning)
- [ ] Assert all form inputs in the access check form have associated `<label>` elements or `aria-label`
- [ ] Assert page has a logical heading hierarchy (h1 > h2 > h3, no skipped levels)
- [ ] Assert all interactive elements are keyboard-focusable with visible focus indicators
- [ ] Assert color contrast ratios meet WCAG AA minimums (4.5:1 for text, 3:1 for large text)
- [ ] Assert tables (if present) have proper `scope` attributes on header cells
- [ ] Assert images and icons have appropriate `alt` text or `aria-hidden="true"`

## Acceptance Criteria

- [ ] axe-core scan returns zero WCAG 2.2 AA violations
- [ ] Approval badges (observe/suggest/execute) include text labels, not color alone
- [ ] Access check form inputs have programmatically associated labels
- [ ] Heading hierarchy is logical and sequential
- [ ] All interactive elements receive keyboard focus and show visible focus indicators
- [ ] Color contrast meets AA minimums across all text and badge elements
- [ ] Tables use `scope="col"` and/or `scope="row"` on header cells

## Definition of Done

- Playwright test file created and passing with axe-core integration
- All accessibility assertions verified
- Test runs green in CI pipeline
