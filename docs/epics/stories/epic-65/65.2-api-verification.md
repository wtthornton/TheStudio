# Story 65.2 — API Endpoint Verification

**Epic:** 65 — Tool Hub: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/tools`

## User Story

As a platform operator, I want the Tool Hub API endpoints to return valid responses so that the frontend can reliably display tool catalog data, profiles, and access check results.

## Description

This story validates the backend API endpoints that power the Tool Hub page. Each endpoint is tested independently for correct HTTP status codes, response structure, and meaningful content. The tests confirm that the tool catalog, profiles, and access check endpoints all return well-formed data.

## Tasks

- [ ] Write Playwright/API test for `GET /admin/tools/catalog` — assert HTTP 200 and response contains tool suite entries
- [ ] Write Playwright/API test for `GET /admin/tools/profiles` — assert HTTP 200 and response contains profile data
- [ ] Write Playwright/API test for `POST /admin/tools/check-access` — assert HTTP 200 and response contains access decision
- [ ] Write Playwright/API test for `GET /admin/ui/partials/tools` — assert HTTP 200 and response contains HTML partial
- [ ] Verify response content types (JSON for API endpoints, HTML for partials)
- [ ] Assert catalog response includes known tool suites (ruff, pytest, security)

## Acceptance Criteria

- [ ] `GET /admin/tools/catalog` returns HTTP 200 with JSON array of tool suites
- [ ] `GET /admin/tools/profiles` returns HTTP 200 with JSON profile data
- [ ] `POST /admin/tools/check-access` returns HTTP 200 with access decision payload
- [ ] `GET /admin/ui/partials/tools` returns HTTP 200 with HTML content
- [ ] Catalog response includes entries for ruff, pytest, and security tool suites
- [ ] No 4xx or 5xx errors from any endpoint under normal conditions

## Definition of Done

- Playwright/API test file created and passing
- All four endpoints tested with status code and response body assertions
- Test runs green in CI pipeline
