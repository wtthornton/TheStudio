# Story 65.4 — Interactive Elements

**Epic:** 65 — Tool Hub: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/tools`

## User Story

As a platform operator, I want interactive elements on the Tool Hub page to function correctly so that I can expand tool suite details, check access for role/tier combinations, and interact with the tool catalog.

## Description

This story validates all interactive elements on the Tool Hub page. Tests exercise click handlers, form submissions, expandable sections, and any dynamic content loading. The access check form is tested end-to-end: filling inputs, submitting, and verifying the result display.

## Tasks

- [ ] Test tool suite card/row expansion — click a tool suite entry and verify details expand or a detail view appears
- [ ] Test access check form — fill role and tier inputs, submit, and verify result is displayed
- [ ] Test any promote/demote button functionality if present
- [ ] Verify dynamic content updates without full page reload (HTMX partial swaps)
- [ ] Test keyboard interaction for interactive elements (Enter/Space to activate buttons)
- [ ] Assert loading states appear during async operations

## Acceptance Criteria

- [ ] Tool suite cards/rows respond to click events and show expanded detail or navigate correctly
- [ ] Access check form accepts role and tier input, submits successfully, and displays an access decision result
- [ ] Interactive buttons (promote, check, etc.) trigger visible state changes
- [ ] No JavaScript errors during any interaction
- [ ] Dynamic updates use partial rendering (no full page reloads for in-page actions)
- [ ] All interactive elements are reachable via keyboard

## Definition of Done

- Playwright test file created and passing
- All interactive elements tested with user-flow scenarios
- Test runs green in CI pipeline
