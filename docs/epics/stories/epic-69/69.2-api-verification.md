# Story 69.2 — API Endpoint Verification

**Epic:** 69 — Dead-Letter Inspector: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want all dead-letter API endpoints to return correct HTTP responses so that the UI can reliably list, inspect, and delete dead-letter events.

## Description

Verify each dead-letter API endpoint responds with the expected status codes and content types. The list partial, detail partial, and delete action must all behave correctly under normal and error conditions.

## APIs Under Test

| Method | Endpoint | Expected |
|--------|----------|----------|
| GET | `/admin/ui/partials/dead-letters` | 200, HTML partial |
| GET | `/admin/ui/partials/dead-letter/{id}` | 200 (valid id) / 404 (invalid id) |
| DELETE | `/admin/ui/partials/dead-letter/{id}` | 200 or 204 on success |

## Tasks

- [ ] Write Playwright test (or API-level test) for `GET /admin/ui/partials/dead-letters` — assert 200 and HTML content
- [ ] Write test for `GET /admin/ui/partials/dead-letter/{id}` with valid ID — assert 200
- [ ] Write test for `GET /admin/ui/partials/dead-letter/{id}` with invalid ID — assert 404 or error partial
- [ ] Write test for `DELETE /admin/ui/partials/dead-letter/{id}` — assert success response
- [ ] Assert response content-types are `text/html` for partials

## Acceptance Criteria

- [ ] All three endpoints return expected status codes for valid requests
- [ ] Invalid ID returns 404 or appropriate error
- [ ] Delete action returns success indicator
- [ ] All responses are HTML partials (content-type text/html)

## Definition of Done

- [ ] Playwright / API tests written and passing
- [ ] Test committed to `tests/playwright/dead-letters/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-69 tracker
