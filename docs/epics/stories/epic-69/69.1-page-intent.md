# Story 69.1 — Page Intent & Semantic Content

**Epic:** 69 — Dead-Letter Inspector: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Dead-Letter Inspector page to clearly communicate its purpose through semantic HTML and meaningful content so that I can immediately understand I am viewing unrecoverable failure events and their details.

## Description

Verify that the `/admin/ui/dead-letters` page renders with correct semantic structure, conveys the "Dead-Letter" heading, displays the event list table or an appropriate empty state, shows failure reasons for each event, and displays attempt count or retry information. The expected vocabulary includes: reason, failure, error, attempt, retry, retries, count, tries.

## Tasks

- [ ] Write Playwright test navigating to `/admin/ui/dead-letters`
- [ ] Assert page contains an `<h1>` or prominent heading with "Dead-Letter"
- [ ] Assert semantic landmark structure (`<main>`, `<nav>`, `<header>` as applicable)
- [ ] Assert event list renders as a `<table>` with appropriate column headers
- [ ] Assert empty state displays "No dead-letter events" when no data present
- [ ] Assert failure reason is displayed for each event in the list (reason, failure, error)
- [ ] Assert attempt count or retry info is visible per event (attempt, retry, retries, count, tries)
- [ ] Validate no broken heading hierarchy (h1 > h2 > h3 in order)

## Acceptance Criteria

- [ ] Playwright test navigates to the dead-letters page and asserts heading text
- [ ] Semantic landmarks are present and correct
- [ ] Table structure renders with column headers or empty state is shown
- [ ] Failure reason is visible for each event
- [ ] Attempt count / retry information is visible for each event
- [ ] Heading hierarchy has no skipped levels

## Definition of Done

- [ ] Playwright test written and passing
- [ ] Test committed to `tests/playwright/dead-letters/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-69 tracker
