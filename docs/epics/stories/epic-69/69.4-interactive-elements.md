# Story 69.4 — Interactive Elements

**Epic:** 69 — Dead-Letter Inspector: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Dead-Letter Inspector interactive elements to function correctly so that I can expand event details, delete dead-letter events with confirmation, and have the UI update without full page reloads.

## Description

Verify that all interactive elements on `/admin/ui/dead-letters` work as expected: table rows are expandable to show event details, the Delete button requires confirmation before acting, and HTMX partial updates refresh content without full page navigation.

## Tasks

- [ ] Write Playwright test for row expansion — click a table row and assert detail panel/content appears
- [ ] Assert detail panel shows failure reason, attempt count, and event metadata
- [ ] Write test for Delete button — click Delete and assert confirmation dialog/prompt appears
- [ ] Write test for Delete confirmation — confirm deletion and assert event is removed from list
- [ ] Write test for Delete cancellation — cancel deletion and assert event remains
- [ ] Assert HTMX partial updates occur (no full page navigation on row expand/delete)
- [ ] Assert UI state is consistent after interaction (counts update, list refreshes)

## Acceptance Criteria

- [ ] Table rows expand to show detail content
- [ ] Detail panel displays failure reason and attempt count
- [ ] Delete button shows confirmation before deleting
- [ ] Confirmed deletion removes the event from the list
- [ ] Cancelled deletion leaves the event in place
- [ ] Interactions use HTMX partial updates (no full page reload)

## Definition of Done

- [ ] Playwright interaction tests written and passing
- [ ] Test committed to `tests/playwright/dead-letters/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-69 tracker
