# Story 69.6 — Visual Snapshot Baseline

**Epic:** 69 — Dead-Letter Inspector: Playwright Full-Stack Test Suite
**Points:** 2
**Status:** Draft

## User Story

As a developer, I want visual snapshot baselines for the Dead-Letter Inspector page so that future changes are caught by visual regression tests and unintended UI drift is prevented.

## Description

Capture Playwright visual snapshots of the `/admin/ui/dead-letters` page in its key states: populated event list, empty state, expanded detail view, and post-delete state. These snapshots serve as the baseline for visual regression detection in CI.

## Tasks

- [ ] Capture full-page snapshot of the dead-letters page with populated event list
- [ ] Capture full-page snapshot of the dead-letters page in empty state ("No dead-letter events")
- [ ] Capture snapshot of a dead-letter detail view (row expanded with failure reason and attempt count)
- [ ] Capture snapshot of the page after a delete action (event removed)
- [ ] Commit all snapshot baselines to the repository
- [ ] Verify snapshots are deterministic (same result on 3 consecutive runs)

## Acceptance Criteria

- [ ] Baseline snapshots captured for: populated list, empty state, expanded detail, post-delete
- [ ] Snapshots are committed to the repository under the test snapshot directory
- [ ] Snapshot comparison passes on re-run (no false-positive diffs)
- [ ] Snapshots capture the full visible viewport

## Definition of Done

- [ ] Playwright visual snapshot tests written and passing
- [ ] Baseline images committed to `tests/playwright/dead-letters/__snapshots__/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-69 tracker
