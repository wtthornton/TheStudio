# Story 59.1 — Fleet Dashboard: Page Intent & Semantic Content

**Epic:** [Epic 59 — Fleet Dashboard Test Suite](../../epic-59-dashboard-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator viewing the Fleet Dashboard page,
**I want** automated tests confirming the page delivers its intended purpose,
**So that** regressions in page content or structure are caught before deployment.

## Description

Validate that `/admin/ui/dashboard` delivers its documented purpose: system health (Temporal, Postgres), workflow metrics (Running/Stuck/Failed/Queue), and repo activity table. Tests verify the page loads with correct heading, contains all required semantic content, and communicates its purpose through domain-specific vocabulary and data structures.

## Tasks

- [ ] Test page loads with 200 and correct heading "Fleet Dashboard"
- [ ] Test system health section shows Temporal and Postgres with status indicators and latency (ms)
- [ ] Test workflow summary section shows Running, Stuck, Failed, and Queue metrics
- [ ] Test repo activity table shows repos or "No repos" empty state
- [ ] Test no HTML entity artifacts (`&#`) in visible text
- [ ] Test no rendering artifacts (NaN, undefined, null)
- [ ] Test no console errors

## Acceptance Criteria

- [ ] Page returns 200 with heading containing "Fleet Dashboard"
- [ ] System health section contains Temporal and Postgres entries with status indicators and latency values
- [ ] Workflow summary contains Running, Stuck, Failed, and Queue metric cards
- [ ] Repo activity table is present with data rows or empty state message
- [ ] Zero console errors during test execution
- [ ] No rendering artifacts in visible text

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
