# Story 59.4 — Fleet Dashboard: Interactive Elements

**Epic:** [Epic 59](../../epic-59-dashboard-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator,
**I want** every button, form, filter, and interactive control on the Fleet Dashboard page tested,
**So that** broken interactions are caught before deployment.

## Description

Test all interactive elements on `/admin/ui/dashboard`: repo activity table row clicks, HTMX-driven dashboard refresh partial. Verify clicks produce expected responses, HTMX swaps update the right DOM targets, and interactive elements respond correctly.

## Tasks

- [ ] Test repo activity table rows are clickable and navigate to repo detail page
- [ ] Test HTMX dashboard content partial refresh triggers correctly
- [ ] Test HTMX swap updates the correct DOM target element
- [ ] Test KPI cards respond to hover/focus states
- [ ] Test health status indicators update without full page reload
- [ ] Test no console errors during any interaction

## Acceptance Criteria

- [ ] All buttons produce visible feedback on click
- [ ] Repo activity table rows navigate to correct detail pages
- [ ] HTMX-driven updates swap correct target elements
- [ ] No console errors during interactions

## Definition of Done

- [ ] All interaction tests pass
- [ ] Uses shared helpers from Epic 58.5
