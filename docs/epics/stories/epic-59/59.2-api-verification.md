# Story 59.2 — Fleet Dashboard: API Endpoint Verification

**Status:** Proposed

---

## User Story

**As a** test engineer,
**I want** every API endpoint backing the Fleet Dashboard page verified,
**So that** the page's data layer is validated independently of rendering.

## Description

Verify all API endpoints that power the Fleet Dashboard page return correct HTTP status codes and valid JSON responses with expected fields.

## Tasks

- [ ] Test `GET /admin/health` returns 200 with JSON containing Temporal, NATS, and DB health status fields
- [ ] Test `GET /admin/workflows/metrics` returns 200 with JSON containing workflow metric counts (running, stuck, failed, queued)
- [ ] Test `GET /admin/repos` returns 200 with JSON array of repo objects
- [ ] Test `GET /admin/ui/partials/dashboard` returns 200 with HTML content (HTMX partial)
- [ ] Test all endpoints return proper Content-Type headers
- [ ] Test error responses do not contain stack traces or internal details

## Acceptance Criteria

- [ ] All endpoints return 200 (or appropriate status) with valid JSON
- [ ] Response bodies contain expected fields
- [ ] No 4xx/5xx errors or traceback content
- [ ] Auth headers correctly forwarded

## Definition of Done

- [ ] All API tests pass against running stack
