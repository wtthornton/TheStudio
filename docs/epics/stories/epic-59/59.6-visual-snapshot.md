# Story 59.6 — Fleet Dashboard: Visual Snapshot Baseline

**Epic:** [Epic 59](../../epic-59-dashboard-playwright-suite.md)
**Points:** 2 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** developer making UI changes,
**I want** a visual snapshot baseline for the Fleet Dashboard page,
**So that** unintended visual regressions are caught by pixel comparison.

## Description

Capture a full-page visual snapshot of `/admin/ui/dashboard` at 1280x720 viewport and store as the baseline. Subsequent test runs compare against this baseline and fail if the pixel diff exceeds the configured threshold (default 0.1%).

## Tasks

- [ ] Capture full-page snapshot of `/admin/ui/dashboard` at 1280x720
- [ ] Capture element-level snapshot of system health section
- [ ] Capture element-level snapshot of workflow KPI cards section
- [ ] Capture element-level snapshot of repo activity table
- [ ] Store baseline in `tests/playwright/snapshots/dashboard/`
- [ ] Configure comparison threshold (0.1% default)
- [ ] Test that intentional changes are detected

## Acceptance Criteria

- [ ] Full-page baseline captured at 1280x720 viewport
- [ ] Baseline stored in correct snapshot directory
- [ ] Comparison passes when page unchanged
- [ ] Comparison fails when page modified beyond threshold
- [ ] Platform-aware naming prevents cross-platform false positives

## Definition of Done

- [ ] Baseline committed
- [ ] Snapshot comparison test passes
- [ ] Uses shared framework from Epic 58.7
