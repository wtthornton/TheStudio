# Story 59.5 — Fleet Dashboard: Accessibility (WCAG 2.2 AA)

**Epic:** [Epic 59](../../epic-59-dashboard-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** user relying on assistive technology,
**I want** the Fleet Dashboard page to meet WCAG 2.2 AA standards,
**So that** the page is usable with screen readers and keyboard-only navigation.

## Description

Validate `/admin/ui/dashboard` meets WCAG 2.2 AA: focus indicators on all interactive elements, logical keyboard tab order, ARIA landmarks and roles, table header scopes, minimum touch targets, color contrast ratios, and non-color state indicators.

## Tasks

- [ ] Test focus-visible ring (2px solid blue) appears on all interactive elements via Tab
- [ ] Test keyboard tab order is logical (matches visual layout)
- [ ] Test ARIA landmarks: `<nav>`, `<main>`, `<header>` present
- [ ] Test KPI metric numbers use correct heading level for screen reader hierarchy
- [ ] Test repo activity table has `scope="col"` on all header cells
- [ ] Test status badges have text labels not just color (e.g., "Healthy" text alongside green indicator)
- [ ] Test minimum touch targets (24x24px) on all buttons/links
- [ ] Run axe-core audit, assert zero critical/serious violations
- [ ] Test status indicators use text + color (not color alone)

## Acceptance Criteria

- [ ] All interactive elements show focus ring on keyboard navigation
- [ ] Tab order matches visual layout
- [ ] All ARIA landmarks present and labeled
- [ ] KPI numbers use semantically correct heading levels
- [ ] Repo activity table headers have `scope="col"`
- [ ] Status badges include text labels alongside color indicators
- [ ] axe-core reports zero critical/serious violations
- [ ] All status indicators have non-color cues

## Definition of Done

- [ ] All a11y tests pass
- [ ] Uses shared helpers from Epic 58.6
