# Story 72.2 — API Endpoint Verification

**Epic:** 72 — Cost Dashboard: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want all Cost Dashboard API endpoints to return valid responses so that the cost UI can reliably load spend data, budget utilization, and provider/model information.

## Description

Verify that all Cost Dashboard-related API endpoints return expected HTTP status codes, correct content types, and valid response bodies. This covers the cost dashboard partial, model spend, budget utilization, providers, and audit trail endpoints.

## Tasks

- [ ] Write Playwright API test for `GET /admin/ui/partials/cost-dashboard` — assert 200 and HTML partial
- [ ] Write Playwright API test for `GET /admin/ui/partials/model-spend` — assert 200 and HTML partial or JSON
- [ ] Write Playwright API test for `GET /admin/ui/partials/budget-utilization` — assert 200 and HTML partial or JSON
- [ ] Write Playwright API test for `GET /admin/models/providers` — assert 200 and JSON response with provider array
- [ ] Write Playwright API test for `GET /admin/models/audit` — assert 200 and JSON response with audit entries
- [ ] Assert provider list contains expected fields (name, model count, cost info)
- [ ] Assert audit trail contains expected fields (timestamp, model, tokens, cost)
- [ ] Assert no endpoint returns 500 under normal conditions

## Acceptance Criteria

- [ ] All 5 endpoints return expected status codes
- [ ] GET endpoints return correct content types (JSON or HTML)
- [ ] Provider list contains structured provider data
- [ ] Audit trail contains structured usage entries
- [ ] No 500 errors under normal conditions

## Definition of Done

- [ ] Playwright API tests written and passing
- [ ] Test committed to `tests/playwright/cost-dashboard/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-72 tracker
