# Story 72.1 — Page Intent & Semantic Content

**Epic:** 72 — Cost Dashboard: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Cost Dashboard page to clearly communicate its purpose through semantic HTML and meaningful content so that I can immediately understand I am viewing LLM cost data, budget utilization, and provider/model breakdowns.

## Description

Verify that the `/admin/ui/cost-dashboard` page renders with correct semantic structure, conveys the "Cost Dashboard" heading, displays cost/spend data with dollar amounts, shows budget utilization metrics, and presents provider and model breakdowns. The page should use appropriate domain vocabulary (spend, budget, utilization, provider, model, cost).

## Tasks

- [ ] Write Playwright test navigating to `/admin/ui/cost-dashboard`
- [ ] Assert page contains an `<h1>` or prominent heading with "Cost Dashboard" or "Cost"
- [ ] Assert semantic landmark structure (`<main>`, `<nav>`, `<header>` as applicable)
- [ ] Assert KPI cards are present showing cost/spend metrics ($ amounts or numeric values)
- [ ] Assert budget utilization is visible (percentage, bar, or progress indicator)
- [ ] Assert provider breakdown section is present (table, list, or chart)
- [ ] Assert model usage breakdown section is present
- [ ] Assert domain vocabulary tokens appear in rendered content (spend, budget, utilization, provider, model, cost)
- [ ] Assert empty state displays appropriate message when no cost data present
- [ ] Validate no broken heading hierarchy (h1 > h2 > h3 in order)

## Acceptance Criteria

- [ ] Playwright test navigates to the cost dashboard page and asserts heading text
- [ ] Semantic landmarks are present and correct
- [ ] KPI cards render with cost/spend metrics
- [ ] Budget utilization is visible
- [ ] Provider and model breakdowns are present
- [ ] Domain vocabulary tokens are present in page content
- [ ] Heading hierarchy has no skipped levels

## Definition of Done

- [ ] Playwright test written and passing
- [ ] Test committed to `tests/playwright/cost-dashboard/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-72 tracker
