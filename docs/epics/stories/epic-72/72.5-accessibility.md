# Story 72.5 — Accessibility WCAG 2.2 AA

**Epic:** 72 — Cost Dashboard: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator using assistive technology, I want the Cost Dashboard page to be fully accessible so that I can understand cost data, navigate tables, and use filters via screen reader or keyboard alone.

## Description

Verify that the `/admin/ui/cost-dashboard` page meets WCAG 2.2 AA requirements. This includes axe-core scan with zero violations, accessible currency formatting, chart text alternatives, progress bar ARIA attributes, and keyboard operability for all interactive elements.

## Tasks

- [ ] Run axe-core accessibility scan on the cost dashboard page — assert zero violations
- [ ] Assert currency values use `font-mono` and are readable by screen readers (no raw symbol issues)
- [ ] Assert charts/visualizations have text alternatives (`aria-label`, visible legend, or `<title>`)
- [ ] Assert progress bars have `aria-valuenow`, `aria-valuemin`, `aria-valuemax` attributes
- [ ] Assert progress bars have `role="progressbar"` or `role="meter"`
- [ ] Assert tables have `scope="col"` on column headers and `scope="row"` where appropriate
- [ ] Assert filter controls (time range, provider) have associated labels or `aria-label`
- [ ] Assert keyboard navigation: Tab key moves through interactive elements in logical order
- [ ] Assert keyboard navigation: Enter/Space activates buttons and selectors
- [ ] Assert focus indicators are visible on all focusable elements
- [ ] Assert color contrast ratios meet 4.5:1 minimum for text
- [ ] Assert color is not the sole means of conveying information (e.g., utilization thresholds use text + color)

## Acceptance Criteria

- [ ] axe-core reports zero violations
- [ ] Currency values are screen-reader friendly
- [ ] Charts have text alternatives
- [ ] Progress bars have correct ARIA attributes
- [ ] Tables have proper scope attributes
- [ ] Keyboard navigation works for all interactive elements
- [ ] Color contrast meets WCAG AA minimums

## Definition of Done

- [ ] Playwright accessibility tests written and passing
- [ ] Test committed to `tests/playwright/cost-dashboard/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-72 tracker
