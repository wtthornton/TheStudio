# Story 72.4 — Interactive Elements

**Epic:** 72 — Cost Dashboard: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want all interactive elements on the Cost Dashboard to function correctly so that I can filter by time range, filter by provider, and drill down into model-level cost details.

## Description

Verify that all interactive elements on the `/admin/ui/cost-dashboard` page respond correctly to user input. This includes time range selectors, provider filters, and possible drill-down interactions to model-level detail views.

## Tasks

- [ ] Write Playwright test for time range selector — assert changing range updates displayed cost data
- [ ] Write Playwright test for provider filter — assert filtering by provider updates the breakdown view
- [ ] Assert drill-down interaction (if present): clicking a provider or model row shows detail
- [ ] Assert KPI cards update when filters are applied
- [ ] Assert table sorting works if column headers are clickable
- [ ] Assert HTMX partial loads correctly when filters trigger content refresh
- [ ] Assert loading indicators appear during data refresh
- [ ] Assert no JavaScript errors in console during interactions
- [ ] Assert filter state is reflected in the UI (selected range, active filter indicators)

## Acceptance Criteria

- [ ] Time range selector updates cost data when changed
- [ ] Provider filter narrows the displayed data
- [ ] Drill-down interactions navigate to or display detail views
- [ ] Filter state is visually indicated
- [ ] No JavaScript errors during interactions

## Definition of Done

- [ ] Playwright interaction tests written and passing
- [ ] Test committed to `tests/playwright/cost-dashboard/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-72 tracker
