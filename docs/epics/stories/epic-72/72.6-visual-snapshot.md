# Story 72.6 — Visual Snapshot Baseline

**Epic:** 72 — Cost Dashboard: Playwright Full-Stack Test Suite
**Points:** 2
**Status:** Draft

## User Story

As a platform operator, I want visual snapshot baselines for the Cost Dashboard page so that any future UI regressions are caught automatically by comparing against the approved baseline.

## Description

Capture and commit Playwright visual snapshots for the `/admin/ui/cost-dashboard` page. Snapshots should cover the main dashboard view, provider breakdown, and model usage sections. Snapshots are taken at a consistent viewport size and compared pixel-by-pixel in CI to catch unintended visual changes.

## Tasks

- [ ] Configure Playwright snapshot settings (viewport: 1280x720, threshold, maxDiffPixels)
- [ ] Capture baseline screenshot for Cost Dashboard main view (with data)
- [ ] Capture baseline screenshot for provider breakdown section
- [ ] Capture baseline screenshot for model usage section
- [ ] Capture baseline screenshot for empty state (if applicable)
- [ ] Capture baseline screenshot with a filter applied (e.g., specific provider selected)
- [ ] Commit all baseline `.png` files to the snapshot directory
- [ ] Verify snapshot comparison passes on re-run (no false positives)

## Acceptance Criteria

- [ ] Baseline screenshots captured for main view and key sections
- [ ] Snapshots taken at consistent viewport size (1280x720)
- [ ] Snapshot comparison passes on consecutive runs (deterministic)
- [ ] Baseline `.png` files committed to repository

## Definition of Done

- [ ] Playwright visual snapshot tests written and passing
- [ ] Baseline screenshots committed to `tests/playwright/cost-dashboard/__screenshots__/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-72 tracker
