# Story 66.2 — API Endpoint Verification

**Epic:** 66 — Model Gateway: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/models`

## User Story

As a platform operator, I want the Model Gateway API endpoints to return valid responses so that the frontend can reliably display model provider data, audit trails, and spend information.

## Description

This story validates the backend API endpoints that power the Model Gateway page. Each endpoint is tested independently for correct HTTP status codes, response structure, and meaningful content. The tests confirm that model providers, audit data, model partials, and spend data endpoints all return well-formed data.

## Tasks

- [ ] Write Playwright/API test for `GET /admin/models/providers` — assert HTTP 200 and response contains provider entries
- [ ] Write Playwright/API test for `GET /admin/models/audit` — assert HTTP 200 and response contains audit trail data
- [ ] Write Playwright/API test for `GET /admin/ui/partials/models` — assert HTTP 200 and response contains HTML partial
- [ ] Write Playwright/API test for `GET /admin/ui/partials/model-spend` — assert HTTP 200 and response contains spend data
- [ ] Verify response content types (JSON for API endpoints, HTML for partials)
- [ ] Assert providers response includes known providers (Anthropic, OpenAI)

## Acceptance Criteria

- [ ] `GET /admin/models/providers` returns HTTP 200 with JSON array of model providers
- [ ] `GET /admin/models/audit` returns HTTP 200 with JSON audit trail entries
- [ ] `GET /admin/ui/partials/models` returns HTTP 200 with HTML content
- [ ] `GET /admin/ui/partials/model-spend` returns HTTP 200 with spend data (HTML or JSON)
- [ ] Providers response includes entries for Anthropic and OpenAI (or equivalent)
- [ ] No 4xx or 5xx errors from any endpoint under normal conditions

## Definition of Done

- Playwright/API test file created and passing
- All four endpoints tested with status code and response body assertions
- Test runs green in CI pipeline
