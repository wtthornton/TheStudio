# Story 66.5 — Accessibility WCAG 2.2 AA

**Epic:** 66 — Model Gateway: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/models`

## User Story

As a platform operator using assistive technology, I want the Model Gateway page to meet WCAG 2.2 AA standards so that I can navigate and understand model providers, routing rules, and cost data using screen readers, keyboard navigation, and other assistive tools.

## Description

This story validates WCAG 2.2 AA compliance for the Model Gateway page. Tests cover semantic HTML structure, ARIA attributes, keyboard navigation, color contrast, and table accessibility. Special attention is given to cost values, model class badges (which must convey meaning through text, not color alone), and table header scoping.

## Tasks

- [ ] Run axe-core accessibility scan on the page and assert zero violations at AA level
- [ ] Assert model class badges use text labels in addition to any color differentiation
- [ ] Assert all tables have `scope="col"` on column headers and `scope="row"` where applicable
- [ ] Assert page has a logical heading hierarchy (h1 > h2 > h3, no skipped levels)
- [ ] Assert all interactive elements are keyboard-focusable with visible focus indicators
- [ ] Assert color contrast ratios meet WCAG AA minimums (4.5:1 for text, 3:1 for large text)
- [ ] Assert cost values in monospace font maintain sufficient contrast
- [ ] Assert form inputs (if any) have programmatically associated labels

## Acceptance Criteria

- [ ] axe-core scan returns zero WCAG 2.2 AA violations
- [ ] Model class badges (fast/balanced/strong) include text labels, not color alone
- [ ] All tables use proper `scope` attributes on header cells
- [ ] Heading hierarchy is logical and sequential
- [ ] All interactive elements receive keyboard focus and show visible focus indicators
- [ ] Color contrast meets AA minimums across all text, badges, and cost values
- [ ] Spend chart or visualization has appropriate text alternatives

## Definition of Done

- Playwright test file created and passing with axe-core integration
- All accessibility assertions verified
- Test runs green in CI pipeline
