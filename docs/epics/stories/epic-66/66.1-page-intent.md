# Story 66.1 — Page Intent & Semantic Content

**Epic:** 66 — Model Gateway: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/models`
**Heading:** "Model Gateway"

## User Story

As a platform operator, I want the Model Gateway page to render with the correct heading, semantic structure, and content so that I can verify model providers, model classes, routing rules, and cost information are displayed correctly.

## Description

This story validates that the Model Gateway page at `/admin/ui/models` loads successfully, displays the "Model Gateway" heading, and contains the expected semantic content. The test navigates to the page, confirms the heading, and verifies that model providers (Anthropic, OpenAI), model classes (fast/balanced/strong), routing rules, and cost information are all present.

## Tasks

- [ ] Write Playwright test that navigates to `/admin/ui/models`
- [ ] Assert page contains an `<h1>` or primary heading with text "Model Gateway"
- [ ] Assert model providers are listed (Anthropic, OpenAI, or similar)
- [ ] Assert model classes are shown (fast, balanced, strong)
- [ ] Assert routing rules are visible mapping pipeline steps to model classes
- [ ] Assert cost information is displayed ($, cost, rate, or equivalent tokens)
- [ ] Verify page returns HTTP 200 and does not show error states

## Acceptance Criteria

- [ ] Test navigates to `/admin/ui/models` and page loads without errors
- [ ] Page heading "Model Gateway" is visible and semantically correct (h1 or equivalent)
- [ ] Model providers listed (at minimum Anthropic, OpenAI)
- [ ] Model classes displayed (fast, balanced, strong)
- [ ] Routing rules section present showing pipeline-stage-to-model-class mapping
- [ ] Cost or spend information visible on the page
- [ ] No console errors or uncaught exceptions during page load

## Definition of Done

- Playwright test file created and passing
- Test covers all acceptance criteria
- Test runs green in CI pipeline
