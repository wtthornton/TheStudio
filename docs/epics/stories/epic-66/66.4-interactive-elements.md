# Story 66.4 — Interactive Elements

**Epic:** 66 — Model Gateway: Playwright Full-Stack Test Suite
**Points:** 3
**Page:** `/admin/ui/models`

## User Story

As a platform operator, I want interactive elements on the Model Gateway page to function correctly so that I can update providers, edit routing rules, and manage budget controls.

## Description

This story validates all interactive elements on the Model Gateway page. Tests exercise provider update controls, routing rule editing, budget controls, and any dynamic content loading. Each interaction is tested end-to-end: triggering the action, verifying state changes, and confirming visual feedback.

## Tasks

- [ ] Test provider update controls — interact with provider settings and verify state change
- [ ] Test routing rule edit functionality — modify a routing rule and verify update
- [ ] Test budget controls — interact with budget or spend limit controls if present
- [ ] Verify dynamic content updates without full page reload (HTMX partial swaps)
- [ ] Test keyboard interaction for interactive elements (Enter/Space to activate buttons)
- [ ] Assert loading states appear during async operations
- [ ] Test any toggle or switch controls for model enable/disable

## Acceptance Criteria

- [ ] Provider update controls respond to interaction and reflect changes
- [ ] Routing rule edit triggers visible state change or confirmation
- [ ] Budget controls accept input and update displayed values
- [ ] No JavaScript errors during any interaction
- [ ] Dynamic updates use partial rendering (no full page reloads for in-page actions)
- [ ] All interactive elements are reachable via keyboard
- [ ] Loading/saving indicators appear during async operations

## Definition of Done

- Playwright test file created and passing
- All interactive elements tested with user-flow scenarios
- Test runs green in CI pipeline
