# Story 66.6 — Visual Snapshot Baseline

**Epic:** 66 — Model Gateway: Playwright Full-Stack Test Suite
**Points:** 2
**Page:** `/admin/ui/models`

## User Story

As a platform operator, I want visual snapshot baselines for the Model Gateway page so that unintended visual regressions are caught automatically in CI.

## Description

This story captures Playwright visual snapshots of the Model Gateway page in its default state and key interactive states. Snapshots serve as the baseline for future regression detection. Any future visual change to the page will require explicit baseline updates, preventing accidental UI drift.

## Tasks

- [ ] Capture full-page screenshot of `/admin/ui/models` in default/loaded state
- [ ] Capture screenshot of model providers section showing provider entries
- [ ] Capture screenshot of routing rules table
- [ ] Capture screenshot of cost/spend display area
- [ ] Store all snapshots in the designated visual baseline directory
- [ ] Configure snapshot comparison threshold (allow minor anti-aliasing differences)

## Acceptance Criteria

- [ ] Full-page snapshot captured and committed as baseline
- [ ] Provider section snapshot captured showing provider cards/rows
- [ ] Routing rules table snapshot captured
- [ ] Cost/spend display snapshot captured
- [ ] Snapshots are deterministic — repeated runs produce matching images
- [ ] Snapshot comparison threshold configured to avoid false positives from anti-aliasing
- [ ] All baselines stored in the project's visual snapshot directory

## Definition of Done

- Playwright visual snapshot test file created and passing
- Baseline images committed to repository
- Snapshot tests run green in CI pipeline
