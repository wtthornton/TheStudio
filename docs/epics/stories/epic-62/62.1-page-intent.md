# Story 62.1 — Audit Log: Page Intent & Semantic Content

**Epic:** [Epic 62 — Audit Log Test Suite](../../epic-62-audit-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator viewing the Audit Log page,
**I want** automated tests confirming the page delivers its intended purpose,
**So that** regressions in page content or structure are caught before deployment.

## Description

Validate that `/admin/ui/audit` delivers its documented purpose: a governance event log for all admin actions supporting forensics and compliance. Tests verify the page loads with the correct heading, contains event type references (registered, updated, changed, paused, resumed), shows time-range filtering options (1h, 6h, 24h, 7d), displays actor and target columns, and shows an events table or empty state.

## Tasks

- [ ] Test page loads with 200 and correct heading "Audit Log"
- [ ] Test event type references are visible (registered, updated, changed, paused, resumed)
- [ ] Test time-range filtering options are available (1h, 6h, 24h, 7d)
- [ ] Test actor and target columns are visible (actor/user and target/repo/resource)
- [ ] Test events table or empty state is shown
- [ ] Test no HTML entity artifacts (`&#`) in visible text
- [ ] Test no rendering artifacts (NaN, undefined, null)
- [ ] Test no console errors

## Acceptance Criteria

- [ ] Page returns 200 with heading containing "Audit Log"
- [ ] Event type references visible in event rows or filter options
- [ ] Time-range filter controls present with 1h, 6h, 24h, 7d options
- [ ] Actor and target columns present in events table header
- [ ] Events table shows data rows or empty state message
- [ ] Zero console errors during test execution
- [ ] No rendering artifacts in visible text

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
