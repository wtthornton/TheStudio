# Story 62.5 — Audit Log: Accessibility WCAG 2.2 AA

**Epic:** [Epic 62 — Audit Log Test Suite](../../epic-62-audit-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** operator who relies on assistive technology,
**I want** the Audit Log page to meet WCAG 2.2 AA standards,
**So that** I can effectively use audit log features regardless of ability.

## Description

Validate that `/admin/ui/audit` meets WCAG 2.2 AA accessibility standards. Tests cover filter control labels, table `scope="col"` attributes, keyboard accessibility for time range buttons, focus indicators, ARIA roles, and contrast ratios.

## Tasks

- [ ] Test filter controls have associated labels (event type filter, time-range selector)
- [ ] Test events table has `scope="col"` on header cells
- [ ] Test time range buttons are keyboard accessible (Tab, Enter, Space)
- [ ] Test visible focus indicators on all interactive elements
- [ ] Test ARIA roles on filter controls and table
- [ ] Test color contrast ratios meet 4.5:1 minimum for text
- [ ] Test page has no accessibility violations via axe-core scan
- [ ] Test screen reader announces filter state changes

## Acceptance Criteria

- [ ] All filter controls have programmatically associated labels
- [ ] Events table header cells have `scope="col"` attribute
- [ ] Time range buttons are fully keyboard accessible
- [ ] Focus indicators visible on all interactive elements with 2px+ outline
- [ ] ARIA roles correctly applied to filter and table regions
- [ ] Text contrast ratios meet 4.5:1 WCAG AA minimum
- [ ] axe-core scan returns zero violations

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
