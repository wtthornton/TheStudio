# Story 62.6 — Audit Log: Visual Snapshot Baseline

**Epic:** [Epic 62 — Audit Log Test Suite](../../epic-62-audit-playwright-suite.md)
**Points:** 2 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** developer modifying the Audit Log page,
**I want** a visual snapshot baseline committed to the repository,
**So that** unintended visual regressions are detected by comparing against the baseline.

## Description

Capture and commit visual snapshot baselines for `/admin/ui/audit` in both light and dark themes. Snapshots cover the full page, the events table, filter controls, and event type badges. These baselines are used for pixel-diff regression detection in CI.

## Tasks

- [ ] Capture full-page snapshot in light theme
- [ ] Capture full-page snapshot in dark theme
- [ ] Capture events table component snapshot
- [ ] Capture filter controls (time-range and event type) component snapshot
- [ ] Capture event type badges component snapshot
- [ ] Capture empty state snapshot (if applicable)
- [ ] Commit baseline images to `tests/playwright/snapshots/audit/`

## Acceptance Criteria

- [ ] Full-page snapshots captured for both light and dark themes
- [ ] Component-level snapshots captured for events table, filter controls, and badges
- [ ] Snapshots committed to `tests/playwright/snapshots/audit/`
- [ ] Snapshot comparison test passes with 0% diff against baseline
- [ ] Empty state snapshot captured if the page supports an empty state

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
