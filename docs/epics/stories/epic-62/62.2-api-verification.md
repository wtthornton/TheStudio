# Story 62.2 — Audit Log: API Endpoint Verification

**Epic:** [Epic 62 — Audit Log Test Suite](../../epic-62-audit-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** test engineer validating the Audit Log page,
**I want** automated tests confirming all backing API endpoints return valid responses,
**So that** API regressions are caught before they break the UI.

## Description

Validate that all API endpoints backing the Audit Log page return 200 with valid JSON. This covers the audit log query endpoint with time range and event type filter support, and the audit partial endpoint used by HTMX.

## Tasks

- [ ] Test `GET /admin/audit` returns 200 with valid JSON
- [ ] Test `GET /admin/audit` supports time range filter parameters
- [ ] Test `GET /admin/audit` supports event type filter parameters
- [ ] Test `GET /admin/ui/partials/audit` returns 200 with valid HTML partial
- [ ] Test `GET /admin/ui/partials/audit` supports filter query parameters
- [ ] Test API responses contain expected fields (event type, actor, target, timestamp)
- [ ] Test no 500 errors on any endpoint

## Acceptance Criteria

- [ ] `GET /admin/audit` returns 200 with valid JSON containing audit event records
- [ ] `GET /admin/audit` correctly accepts time range filter parameters
- [ ] `GET /admin/audit` correctly accepts event type filter parameters
- [ ] `GET /admin/ui/partials/audit` returns 200 with HTML content
- [ ] Response payloads contain expected fields (event_type, actor, target, timestamp)
- [ ] No 500 errors on any tested endpoint

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
