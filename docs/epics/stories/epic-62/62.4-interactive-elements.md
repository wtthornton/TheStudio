# Story 62.4 — Audit Log: Interactive Elements

**Epic:** [Epic 62 — Audit Log Test Suite](../../epic-62-audit-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator using the Audit Log page,
**I want** all interactive elements to function correctly,
**So that** I can filter and navigate audit events without encountering broken controls.

## Description

Validate that all interactive elements on `/admin/ui/audit` function correctly. This includes time-range filter buttons/select changing displayed results, event type filter changing displayed events, and HTMX partial refresh behavior.

## Tasks

- [ ] Test time-range filter buttons/select change displayed results (1h, 6h, 24h, 7d)
- [ ] Test event type filter changes displayed events
- [ ] Test HTMX partial refresh updates the events table without full page reload
- [ ] Test combining time-range and event type filters produces correct results
- [ ] Test filter reset/clear returns to default view
- [ ] Test no console errors during interactions
- [ ] Test interactive elements have appropriate cursor styles

## Acceptance Criteria

- [ ] Clicking time-range filter options updates the events table with filtered results
- [ ] Changing event type filter updates displayed events accordingly
- [ ] HTMX partial refresh works without full page reload
- [ ] Combined filters produce correct filtered results
- [ ] No console errors during any interaction
- [ ] All interactive elements show pointer cursor on hover

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
