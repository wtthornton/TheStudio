# Story 62.3 — Audit Log: Style Guide Compliance

**Epic:** [Epic 62 — Audit Log Test Suite](../../epic-62-audit-playwright-suite.md)
**Points:** 5 | **Size:** M
**Status:** Proposed

---

## User Story

**As a** design-conscious operator viewing the Audit Log page,
**I want** the page to conform to the UI/UX style guide,
**So that** the audit log is visually consistent with the rest of the admin interface.

## Description

Validate that `/admin/ui/audit` conforms to `docs/design/07-THESTUDIO-UI-UX-STYLE-GUIDE.md`. Tests check color tokens, typography, spacing, and component recipes for the data table (audit log), time-range selector, event type filter, and badges for event types.

## Tasks

- [ ] Test page background and surface colors match style guide tokens
- [ ] Test heading typography (font-family, font-size, font-weight) matches Section 4.2
- [ ] Test data table styling matches component recipe (borders, row spacing, header styling)
- [ ] Test event type badges use correct color tokens per Section 5.1
- [ ] Test time-range selector styling matches button/select component recipe
- [ ] Test event type filter styling matches filter component recipe
- [ ] Test spacing between sections matches style guide spacing scale
- [ ] Test both light and dark theme compliance

## Acceptance Criteria

- [ ] Background and surface colors match style guide tokens (light and dark)
- [ ] Typography matches Section 4.2 specifications
- [ ] Data table follows component recipe with correct borders and spacing
- [ ] Event type badges use status color tokens from Section 5.1
- [ ] Time-range selector and event type filter follow component recipes
- [ ] Spacing between sections follows the spacing scale
- [ ] Both light and dark themes pass all checks

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
