# Story 73.1 — Page Intent & Semantic Content

**Epic:** 73 — Portfolio Health: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Portfolio Health page to clearly communicate its purpose through semantic HTML and meaningful content so that I can immediately understand the aggregate health status across all repositories in the portfolio.

## Description

Verify that the `/admin/ui/portfolio-health` page renders with correct semantic structure, conveys the "Portfolio Health" heading, displays aggregate health status (healthy/degraded/unhealthy counts), per-repo health indicators, and risk distribution. The page should use appropriate domain vocabulary (health, healthy, degraded, unhealthy, risk, portfolio, repo, status).

## Tasks

- [ ] Write Playwright test navigating to `/admin/ui/portfolio-health`
- [ ] Assert page contains an `<h1>` or prominent heading with "Portfolio Health"
- [ ] Assert semantic landmark structure (`<main>`, `<nav>`, `<header>` as applicable)
- [ ] Assert summary cards are present showing aggregate health counts (healthy, degraded, unhealthy)
- [ ] Assert per-repo health table or list is present with status indicators
- [ ] Assert risk distribution section is present (chart, badges, or summary)
- [ ] Assert domain vocabulary tokens appear in rendered content (health, healthy, degraded, unhealthy, risk, portfolio, repo, status)
- [ ] Assert empty state displays appropriate message when no repos are configured
- [ ] Validate no broken heading hierarchy (h1 > h2 > h3 in order)

## Acceptance Criteria

- [ ] Playwright test navigates to the portfolio health page and asserts heading text
- [ ] Semantic landmarks are present and correct
- [ ] Summary cards render with aggregate health counts
- [ ] Per-repo health indicators are present
- [ ] Risk distribution is visible
- [ ] Domain vocabulary tokens are present in page content
- [ ] Heading hierarchy has no skipped levels

## Definition of Done

- [ ] Playwright test written and passing
- [ ] Test committed to `tests/playwright/portfolio-health/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-73 tracker
