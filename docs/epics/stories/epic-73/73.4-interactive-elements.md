# Story 73.4 — Interactive Elements

**Epic:** 73 — Portfolio Health: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want all interactive elements on the Portfolio Health page to function correctly so that I can filter repos, filter by health status, and drill down to individual repo details.

## Description

Verify that all interactive elements on the `/admin/ui/portfolio-health` page respond correctly to user input. This includes repo name filtering, health status filtering, and drill-down navigation to repo detail pages.

## Tasks

- [ ] Write Playwright test for repo filter — assert typing a repo name filters the displayed list
- [ ] Write Playwright test for health status filter — assert selecting a status (healthy/degraded/unhealthy) narrows results
- [ ] Assert drill-down interaction: clicking a repo row or link navigates to repo detail page
- [ ] Assert summary cards update when filters are applied (counts reflect filtered set)
- [ ] Assert table sorting works if column headers are clickable
- [ ] Assert HTMX partial loads correctly when filters trigger content refresh
- [ ] Assert loading indicators appear during data refresh
- [ ] Assert no JavaScript errors in console during interactions
- [ ] Assert filter state is reflected in the UI (active filter indicators, search text)
- [ ] Assert clearing filters restores the full repo list

## Acceptance Criteria

- [ ] Repo filter narrows displayed repos by name
- [ ] Health status filter narrows repos by health status
- [ ] Drill-down navigation works to repo detail pages
- [ ] Summary cards update with filtered counts
- [ ] Filter state is visually indicated
- [ ] No JavaScript errors during interactions

## Definition of Done

- [ ] Playwright interaction tests written and passing
- [ ] Test committed to `tests/playwright/portfolio-health/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-73 tracker
