# Story 73.6 — Visual Snapshot Baseline

**Epic:** 73 — Portfolio Health: Playwright Full-Stack Test Suite
**Points:** 2
**Status:** Draft

## User Story

As a platform operator, I want visual snapshot baselines for the Portfolio Health page so that any future UI regressions are caught automatically by comparing against the approved baseline.

## Description

Capture and commit Playwright visual snapshots for the `/admin/ui/portfolio-health` page. Snapshots should cover the main health overview, repo table, risk distribution, and empty state. Snapshots are taken at a consistent viewport size and compared pixel-by-pixel in CI to catch unintended visual changes.

## Tasks

- [ ] Configure Playwright snapshot settings (viewport: 1280x720, threshold, maxDiffPixels)
- [ ] Capture baseline screenshot for Portfolio Health main view (with data)
- [ ] Capture baseline screenshot for repo health table section
- [ ] Capture baseline screenshot for risk distribution section
- [ ] Capture baseline screenshot for empty state (no repos configured)
- [ ] Capture baseline screenshot with a filter applied (e.g., "degraded" status filter)
- [ ] Commit all baseline `.png` files to the snapshot directory
- [ ] Verify snapshot comparison passes on re-run (no false positives)

## Acceptance Criteria

- [ ] Baseline screenshots captured for main view and key sections
- [ ] Snapshots taken at consistent viewport size (1280x720)
- [ ] Snapshot comparison passes on consecutive runs (deterministic)
- [ ] Baseline `.png` files committed to repository

## Definition of Done

- [ ] Playwright visual snapshot tests written and passing
- [ ] Baseline screenshots committed to `tests/playwright/portfolio-health/__screenshots__/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-73 tracker
