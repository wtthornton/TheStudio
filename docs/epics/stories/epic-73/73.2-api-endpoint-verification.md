# Story 73.2 — API Endpoint Verification

**Epic:** 73 — Portfolio Health: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want all Portfolio Health API endpoints to return valid responses so that the health UI can reliably display aggregate status, per-repo health, and risk distribution.

## Description

Verify that all Portfolio Health-related API endpoints return expected HTTP status codes, correct content types, and valid response bodies. This covers the portfolio health partial, repo list, and repo health status endpoints.

## Tasks

- [ ] Write Playwright API test for `GET /admin/ui/partials/portfolio-health` — assert 200 and HTML partial
- [ ] Write Playwright API test for `GET /admin/repos` — assert 200 and JSON response with repo array
- [ ] Write Playwright API test for `GET /admin/repos/health` — assert 200 and JSON response with health data
- [ ] Assert repo list contains expected fields (id, name, status, tier)
- [ ] Assert health data contains expected fields (repo_id, health_status, risk_level or equivalent)
- [ ] Assert health status values are from expected vocabulary (healthy, degraded, unhealthy)
- [ ] Assert no endpoint returns 500 under normal conditions

## Acceptance Criteria

- [ ] All 3 endpoints return expected status codes
- [ ] GET endpoints return correct content types (JSON or HTML)
- [ ] Repo list contains structured repo data
- [ ] Health data contains structured health indicators
- [ ] Health status values use expected vocabulary
- [ ] No 500 errors under normal conditions

## Definition of Done

- [ ] Playwright API tests written and passing
- [ ] Test committed to `tests/playwright/portfolio-health/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-73 tracker
