# Story 73.5 — Accessibility WCAG 2.2 AA

**Epic:** 73 — Portfolio Health: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator using assistive technology, I want the Portfolio Health page to be fully accessible so that I can understand aggregate health status, navigate the repo table, and use filters via screen reader or keyboard alone.

## Description

Verify that the `/admin/ui/portfolio-health` page meets WCAG 2.2 AA requirements. This includes axe-core scan with zero violations, health status badges conveying meaning without relying solely on color, labeled summary numbers, proper table markup, and keyboard operability.

## Tasks

- [ ] Run axe-core accessibility scan on the portfolio health page — assert zero violations
- [ ] Assert health status badges use text + color (not color alone) to convey status
- [ ] Assert summary cards have descriptive labels (e.g., "Healthy: 5", not just "5")
- [ ] Assert tables have `scope="col"` on column headers
- [ ] Assert tables have `scope="row"` on row headers where applicable
- [ ] Assert filter controls have associated labels or `aria-label`
- [ ] Assert keyboard navigation: Tab key moves through interactive elements in logical order
- [ ] Assert keyboard navigation: Enter/Space activates buttons, links, and filters
- [ ] Assert focus indicators are visible on all focusable elements
- [ ] Assert color contrast ratios meet 4.5:1 minimum for text
- [ ] Assert status indicators convey meaning without relying solely on color (text, icon, or pattern supplement)
- [ ] Assert drill-down links have descriptive accessible names (not just "View" but "View repo X")

## Acceptance Criteria

- [ ] axe-core reports zero violations
- [ ] Health status badges are accessible (text + color)
- [ ] Summary numbers have descriptive labels
- [ ] Tables have proper scope attributes
- [ ] Keyboard navigation works for all interactive elements
- [ ] Color contrast meets WCAG AA minimums
- [ ] Status meaning is conveyed without color alone

## Definition of Done

- [ ] Playwright accessibility tests written and passing
- [ ] Test committed to `tests/playwright/portfolio-health/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-73 tracker
