# Story 60.1 — Repo Management: Page Intent & Semantic Content

**Epic:** [Epic 60 — Repo Management Test Suite](../../epic-60-repos-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator viewing the Repo Management page,
**I want** automated tests confirming the page delivers its intended purpose,
**So that** regressions in page content or structure are caught before deployment.

## Description

Validate that `/admin/ui/repos` delivers its documented purpose: register repos, view list with tier/status badges, control processing (pause/resume/writes). Tests verify the page loads with correct heading, contains all required semantic content, and communicates its purpose through domain-specific vocabulary and data structures.

## Tasks

- [ ] Test page loads with 200 and correct heading "Repo Management"
- [ ] Test "Register Repo" button or form is accessible and visible
- [ ] Test repo list table shows repos or empty state with guidance text
- [ ] Test tier badges (shadow/probation/trusted) are displayed with correct colors
- [ ] Test status indicators (active/paused) are visible for each repo
- [ ] Test no HTML entity artifacts (`&#`) in visible text
- [ ] Test no rendering artifacts (NaN, undefined, null)
- [ ] Test no console errors

## Acceptance Criteria

- [ ] Page returns 200 with heading containing "Repo Management"
- [ ] Register Repo action is discoverable on the page
- [ ] Repo list table or empty state is present and correctly structured
- [ ] Tier badges display for each repo with correct tier labels
- [ ] Zero console errors during test execution
- [ ] No rendering artifacts in visible text

## Definition of Done

- [ ] All tests pass against running Docker stack
- [ ] Tests use shared helpers from Epic 58
