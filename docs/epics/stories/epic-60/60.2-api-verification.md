# Story 60.2 — Repo Management: API Endpoint Verification

**Epic:** [Epic 60](../../epic-60-repos-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** test engineer,
**I want** every API endpoint backing the Repo Management page verified,
**So that** the page's data layer is validated independently of rendering.

## Description

Verify all API endpoints that power the Repo Management page return correct HTTP status codes and valid JSON responses with expected fields.

## Tasks

- [ ] Test `GET /admin/repos` returns 200 with JSON array of repo objects containing name, tier, and status fields
- [ ] Test `POST /admin/repos` accepts a valid repo registration payload and returns 200/201 with created repo data
- [ ] Test `POST /admin/repos` with invalid payload returns 4xx with descriptive error
- [ ] Test `GET /admin/repos/health` returns 200 with JSON containing repo health information
- [ ] Test `GET /admin/ui/partials/repos` returns 200 with HTML content (HTMX partial)
- [ ] Test all endpoints return proper Content-Type headers
- [ ] Test error responses do not contain stack traces or internal details

## Acceptance Criteria

- [ ] All endpoints return 200 (or appropriate status) with valid JSON
- [ ] Response bodies contain expected fields
- [ ] No 4xx/5xx errors or traceback content
- [ ] Auth headers correctly forwarded

## Definition of Done

- [ ] All API tests pass against running stack
