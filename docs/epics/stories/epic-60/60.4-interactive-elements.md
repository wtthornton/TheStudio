# Story 60.4 — Repo Management: Interactive Elements

**Epic:** [Epic 60](../../epic-60-repos-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As an** admin operator,
**I want** every button, form, filter, and interactive control on the Repo Management page tested,
**So that** broken interactions are caught before deployment.

## Description

Test all interactive elements on `/admin/ui/repos`: Register Repo button opens form, form submits via HTMX with result shown in #register-result, table rows link to detail page. Verify clicks produce expected responses, forms submit correctly, HTMX swaps update the right DOM targets, and filters/selects modify displayed data.

## Tasks

- [ ] Test Register Repo button is visible and clickable
- [ ] Test clicking Register Repo button opens the registration form
- [ ] Test registration form submits via HTMX (not full page reload)
- [ ] Test successful registration result is shown in `#register-result` target
- [ ] Test registration form validation prevents empty/invalid submissions
- [ ] Test repo table rows link to individual repo detail pages
- [ ] Test Pause/Resume controls toggle repo processing status
- [ ] Test no console errors during any interaction

## Acceptance Criteria

- [ ] All buttons produce visible feedback on click
- [ ] Register Repo form opens, submits, and shows result in correct target
- [ ] HTMX-driven updates swap correct target elements
- [ ] No console errors during interactions

## Definition of Done

- [ ] All interaction tests pass
- [ ] Uses shared helpers from Epic 58.5
