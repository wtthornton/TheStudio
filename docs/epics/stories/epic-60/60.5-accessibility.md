# Story 60.5 — Repo Management: Accessibility (WCAG 2.2 AA)

**Epic:** [Epic 60](../../epic-60-repos-playwright-suite.md)
**Points:** 3 | **Size:** S
**Status:** Proposed

---

## User Story

**As a** user relying on assistive technology,
**I want** the Repo Management page to meet WCAG 2.2 AA standards,
**So that** the page is usable with screen readers and keyboard-only navigation.

## Description

Validate `/admin/ui/repos` meets WCAG 2.2 AA: focus indicators on all interactive elements, logical keyboard tab order, ARIA landmarks and roles, table header scopes, form label associations, minimum touch targets, color contrast ratios, and non-color state indicators.

## Tasks

- [ ] Test focus-visible ring (2px solid blue) appears on all interactive elements via Tab
- [ ] Test keyboard tab order is logical (matches visual layout)
- [ ] Test ARIA landmarks: `<nav>`, `<main>`, `<header>` present
- [ ] Test registration form inputs have associated `<label>` elements
- [ ] Test repo table has `scope="col"` on all header cells
- [ ] Test tier badges have text labels alongside color (not color alone)
- [ ] Test minimum touch targets (24x24px) on all buttons/links
- [ ] Run axe-core audit, assert zero critical/serious violations
- [ ] Test status indicators use text + color (not color alone)

## Acceptance Criteria

- [ ] All interactive elements show focus ring on keyboard navigation
- [ ] Tab order matches visual layout
- [ ] All ARIA landmarks present and labeled
- [ ] Registration form inputs are properly labeled for screen readers
- [ ] Repo table headers have `scope="col"`
- [ ] Tier badges include text labels alongside color indicators
- [ ] axe-core reports zero critical/serious violations
- [ ] All status indicators have non-color cues

## Definition of Done

- [ ] All a11y tests pass
- [ ] Uses shared helpers from Epic 58.6
