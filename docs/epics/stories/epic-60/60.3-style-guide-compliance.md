# Story 60.3 — Repo Management: Style Guide Compliance

**Epic:** [Epic 60](../../epic-60-repos-playwright-suite.md)
**Points:** 5 | **Size:** M
**Status:** Proposed

---

## User Story

**As a** design-systems maintainer,
**I want** automated verification that the Repo Management page conforms to the UI/UX style guide,
**So that** visual inconsistencies are caught automatically.

## Description

Validate that `/admin/ui/repos` conforms to `docs/design/07-THESTUDIO-UI-UX-STYLE-GUIDE.md`: correct status colors (Section 5), typography scale (Section 6), spacing grid (Section 7), layout shell (Section 8), and component recipes (Section 9) for all data table, status badges (tier + status), form (register repo), buttons, and empty state components.

## Tasks

- [ ] Validate sidebar conforms to Section 8.2 (dark bg, 224px width, active item highlight)
- [ ] Validate header bar conforms to Section 8.3 (56px height, border-bottom)
- [ ] Validate repo data table uses correct table recipe per Section 9
- [ ] Validate tier badges (shadow/probation/trusted) use correct status colors per Section 5.1
- [ ] Validate status badges (active/paused) use correct colors per Section 5.1
- [ ] Validate Register Repo button uses correct button recipe per Section 9
- [ ] Validate registration form inputs use correct form field recipe per Section 9
- [ ] Validate empty state component follows empty state recipe per Section 9
- [ ] Validate status badge colors match Section 5.1
- [ ] Validate typography scale (headings, body, labels) per Section 6.2
- [ ] Validate spacing uses 4px grid per Section 7.1

## Acceptance Criteria

- [ ] All components on page match their style guide recipe
- [ ] Status colors validated for all badge/indicator variants on page
- [ ] Typography validated for all heading levels and body text
- [ ] Spacing validated against 4px grid
- [ ] No hardcoded colors outside the token system

## Definition of Done

- [ ] All style tests pass
- [ ] Uses shared validators from Epic 58.3
