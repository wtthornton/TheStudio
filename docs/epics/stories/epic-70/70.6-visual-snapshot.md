# Story 70.6 — Visual Snapshot Baseline

**Epic:** 70 — Execution Planes: Playwright Full-Stack Test Suite
**Points:** 2
**Status:** Draft

## User Story

As a developer, I want visual snapshot baselines for the Execution Planes page so that future changes are caught by visual regression tests and unintended UI drift is prevented.

## Description

Capture Playwright visual snapshots of the `/admin/ui/planes` page in its key states: populated plane list with summary cards, empty state, registration form, and post-action states (after pause/resume). These snapshots serve as the baseline for visual regression detection in CI.

## Tasks

- [ ] Capture full-page snapshot of the planes page with populated plane list and summary cards
- [ ] Capture full-page snapshot of the planes page in empty state (no planes registered)
- [ ] Capture snapshot of the registration form (focused / filled state)
- [ ] Capture snapshot of the page after a pause action (plane showing paused status)
- [ ] Capture snapshot of the page after a resume action (plane showing active status)
- [ ] Commit all snapshot baselines to the repository
- [ ] Verify snapshots are deterministic (same result on 3 consecutive runs)

## Acceptance Criteria

- [ ] Baseline snapshots captured for: populated list, empty state, registration form, paused state, resumed state
- [ ] Snapshots are committed to the repository under the test snapshot directory
- [ ] Snapshot comparison passes on re-run (no false-positive diffs)
- [ ] Snapshots capture the full visible viewport

## Definition of Done

- [ ] Playwright visual snapshot tests written and passing
- [ ] Baseline images committed to `tests/playwright/planes/__snapshots__/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-70 tracker
