# Story 70.3 — Style Guide Compliance

**Epic:** 70 — Execution Planes: Playwright Full-Stack Test Suite
**Points:** 5
**Status:** Draft

## User Story

As a platform operator, I want the Execution Planes page to follow the TheStudio UI/UX Style Guide so that it is visually consistent with the rest of the admin interface and maintains design system integrity.

## Description

Verify that the `/admin/ui/planes` page adheres to `docs/design/07-THESTUDIO-UI-UX-STYLE-GUIDE.md`. This includes correct use of color tokens, typography scale, spacing system, component patterns (summary cards, data table, status badges, registration form, action buttons, empty state), and layout structure.

## Tasks

- [ ] Assert page uses the correct CSS custom properties / design tokens for colors
- [ ] Assert typography follows the style guide scale (headings, body, captions)
- [ ] Assert spacing between sections and components follows the spacing system
- [ ] Assert summary cards follow the card component pattern from the guide
- [ ] Assert data table follows the style guide table pattern (borders, row hover, padding)
- [ ] Assert status badges (active, paused, draining) use correct badge variant styles
- [ ] Assert registration form follows form component styling from the guide (inputs, labels, submit button)
- [ ] Assert action buttons (Pause, Resume) follow button variant rules
- [ ] Assert empty state follows the empty state pattern from the guide
- [ ] Assert page layout follows the admin shell grid / layout pattern

## Acceptance Criteria

- [ ] All color tokens match the style guide
- [ ] Typography scale is correct for headings, body, and labels
- [ ] Spacing is consistent with the design system
- [ ] Summary cards match the style guide card component
- [ ] Data table matches the style guide table component
- [ ] Status badges use correct variant styles
- [ ] Registration form follows the style guide form pattern
- [ ] Empty state follows the documented empty state pattern
- [ ] No invented visual patterns — all patterns from the style guide

## Definition of Done

- [ ] Playwright style compliance tests written and passing
- [ ] Test committed to `tests/playwright/planes/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-70 tracker
