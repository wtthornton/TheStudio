# Story 70.4 — Interactive Elements

**Epic:** 70 — Execution Planes: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Execution Planes interactive elements to function correctly so that I can register new planes, pause and resume existing planes, and see summary counts update in response to changes.

## Description

Verify that all interactive elements on `/admin/ui/planes` work as expected: the registration form submits via HTMX, Pause/Resume buttons toggle plane state, and summary counts update on change. All interactions should use HTMX partial updates without full page reloads.

## Tasks

- [ ] Write Playwright test for registration form — fill in plane details and submit
- [ ] Assert new plane appears in the plane list after successful registration
- [ ] Assert registration form validates required fields (shows error on empty submit)
- [ ] Write test for Pause button — click Pause on an active plane and assert status changes to "paused"
- [ ] Write test for Resume button — click Resume on a paused plane and assert status changes to "active"
- [ ] Assert summary counts update after registration (total count increases)
- [ ] Assert summary counts update after pause/resume (active count changes)
- [ ] Assert HTMX partial updates occur (no full page navigation on form submit or button click)

## Acceptance Criteria

- [ ] Registration form submits and adds a new plane to the list
- [ ] Registration form validates required fields
- [ ] Pause button changes plane status to paused
- [ ] Resume button changes plane status to active
- [ ] Summary counts reflect current state after changes
- [ ] Interactions use HTMX partial updates (no full page reload)

## Definition of Done

- [ ] Playwright interaction tests written and passing
- [ ] Test committed to `tests/playwright/planes/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-70 tracker
