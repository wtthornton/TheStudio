# Story 70.2 — API Endpoint Verification

**Epic:** 70 — Execution Planes: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want all execution plane API endpoints to return correct HTTP responses so that the UI can reliably list planes, register new planes, and pause/resume existing planes.

## Description

Verify each execution plane API endpoint responds with the expected status codes and content types. The list partial, register action, pause action, and resume action must all behave correctly under normal and error conditions.

## APIs Under Test

| Method | Endpoint | Expected |
|--------|----------|----------|
| GET | `/admin/ui/partials/planes` | 200, HTML partial |
| POST | `/admin/ui/partials/planes/register` | 200 or 302 on success |
| POST | `/admin/ui/partials/planes/{id}/pause` | 200 or 302 on success |
| POST | `/admin/ui/partials/planes/{id}/resume` | 200 or 302 on success |

## Tasks

- [ ] Write Playwright test (or API-level test) for `GET /admin/ui/partials/planes` — assert 200 and HTML content
- [ ] Write test for `POST /admin/ui/partials/planes/register` with valid payload — assert success response
- [ ] Write test for `POST /admin/ui/partials/planes/register` with invalid/missing payload — assert validation error
- [ ] Write test for `POST /admin/ui/partials/planes/{id}/pause` — assert success response
- [ ] Write test for `POST /admin/ui/partials/planes/{id}/resume` — assert success response
- [ ] Assert response content-types are `text/html` for partials

## Acceptance Criteria

- [ ] All four endpoints return expected status codes for valid requests
- [ ] Registration with valid data returns success
- [ ] Registration with invalid data returns validation error
- [ ] Pause action returns success indicator
- [ ] Resume action returns success indicator
- [ ] All responses are HTML partials (content-type text/html)

## Definition of Done

- [ ] Playwright / API tests written and passing
- [ ] Test committed to `tests/playwright/planes/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-70 tracker
