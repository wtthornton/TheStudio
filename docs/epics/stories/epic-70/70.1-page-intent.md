# Story 70.1 — Page Intent & Semantic Content

**Epic:** 70 — Execution Planes: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Execution Planes page to clearly communicate its purpose through semantic HTML and meaningful content so that I can immediately understand I am viewing worker clusters, their health, and their assignments.

## Description

Verify that the `/admin/ui/planes` page renders with correct semantic structure, conveys the "Execution Planes" heading, displays summary counts (total, active, planes, repos), a plane list table or appropriate empty state, status indicators (active, paused, draining, health), region information, and a registration form or capability.

## Tasks

- [ ] Write Playwright test navigating to `/admin/ui/planes`
- [ ] Assert page contains an `<h1>` or prominent heading with "Execution Planes"
- [ ] Assert semantic landmark structure (`<main>`, `<nav>`, `<header>` as applicable)
- [ ] Assert summary counts are visible (total, active, planes, repos)
- [ ] Assert plane list renders as a `<table>` with appropriate column headers
- [ ] Assert empty state is displayed when no planes are registered
- [ ] Assert status indicators are present (active, paused, draining, health)
- [ ] Assert region information is shown or accepted in the registration form (region, default)
- [ ] Assert registration form or registration capability is present (form, input, button)
- [ ] Validate no broken heading hierarchy (h1 > h2 > h3 in order)

## Acceptance Criteria

- [ ] Playwright test navigates to the planes page and asserts heading text
- [ ] Semantic landmarks are present and correct
- [ ] Summary counts are visible on the page
- [ ] Table structure renders with column headers or empty state is shown
- [ ] Status indicators are present for each plane
- [ ] Region information is shown or available in the registration form
- [ ] Registration form / capability is present
- [ ] Heading hierarchy has no skipped levels

## Definition of Done

- [ ] Playwright test written and passing
- [ ] Test committed to `tests/playwright/planes/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-70 tracker
