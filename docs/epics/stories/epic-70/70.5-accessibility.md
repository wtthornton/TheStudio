# Story 70.5 — Accessibility WCAG 2.2 AA

**Epic:** 70 — Execution Planes: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator using assistive technology, I want the Execution Planes page to meet WCAG 2.2 AA standards so that I can navigate, understand, and operate all plane management features regardless of ability.

## Description

Verify that `/admin/ui/planes` passes WCAG 2.2 AA accessibility requirements. Run axe-core automated checks, verify keyboard navigability, confirm proper ARIA labels, validate table `scope` attributes, ensure registration form inputs have associated labels, confirm status badges use text + color (not color alone), and verify action buttons have clear accessible labels.

## Tasks

- [ ] Run axe-core accessibility scan via Playwright and assert zero violations
- [ ] Assert all `<th>` elements in the plane table have `scope="col"` or `scope="row"`
- [ ] Assert all registration form inputs have associated `<label>` elements or `aria-label`
- [ ] Assert Pause and Resume buttons have accessible names (visible text or `aria-label`)
- [ ] Assert status badges convey meaning beyond color alone (text labels present: active, paused, draining)
- [ ] Test keyboard navigation: Tab through registration form, table rows, and action buttons
- [ ] Assert focus is visible on all interactive elements (focus ring / outline)
- [ ] Assert color contrast ratios meet AA minimum (4.5:1 for text, 3:1 for large text)
- [ ] Assert summary cards are accessible (semantic headings or `aria-label` for counts)

## Acceptance Criteria

- [ ] axe-core scan reports zero WCAG 2.2 AA violations
- [ ] Table headers have correct `scope` attributes
- [ ] All form inputs have accessible labels
- [ ] All action buttons have accessible names
- [ ] Status badges use text + color (not color alone)
- [ ] Full keyboard navigation works without mouse
- [ ] Focus indicators are visible on all interactive elements
- [ ] Color contrast meets AA thresholds

## Definition of Done

- [ ] Playwright accessibility tests written and passing
- [ ] axe-core scan integrated into test
- [ ] Test committed to `tests/playwright/planes/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-70 tracker
