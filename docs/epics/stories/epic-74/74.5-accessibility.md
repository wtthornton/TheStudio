# Story 74.5 — Accessibility WCAG 2.2 AA

**Epic:** 74 — Detail Pages (Repo/Workflow/Expert): Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator using assistive technology, I want all detail pages to be fully accessible so that I can understand entity details, navigate controls, and perform actions via screen reader or keyboard alone.

## Description

Verify that the Repo, Workflow, and Expert detail pages meet WCAG 2.2 AA requirements. This includes axe-core scan with zero violations on all 3 page types, labeled action buttons, accessible forms, proper table markup, and keyboard operability for all interactive elements.

## Tasks

### Repo Detail Accessibility
- [ ] Run axe-core scan on repo detail page — assert zero violations
- [ ] Assert all action buttons (Pause, Resume) have descriptive accessible names
- [ ] Assert tier change form inputs have associated labels
- [ ] Assert merge mode toggle has an accessible name and conveys state
- [ ] Assert promotion history table has `scope` attributes on headers

### Workflow Detail Accessibility
- [ ] Run axe-core scan on workflow detail page — assert zero violations
- [ ] Assert all action buttons (Rerun Verification, Escalate, Send-to-Agent) have descriptive accessible names
- [ ] Assert pipeline stage indicators are accessible (text + color, not color alone)
- [ ] Assert status information is conveyed through text, not just visual indicators

### Expert Detail Accessibility
- [ ] Run axe-core scan on expert detail page — assert zero violations
- [ ] Assert trust tier, confidence, and weight values have descriptive labels
- [ ] Assert drift analysis visualization has a text alternative
- [ ] Assert any charts/graphs have `aria-label` or visible legend

### Cross-Cutting
- [ ] Assert keyboard navigation: Tab key moves through interactive elements in logical order on all 3 pages
- [ ] Assert keyboard navigation: Enter/Space activates buttons and form controls
- [ ] Assert focus indicators are visible on all focusable elements
- [ ] Assert color contrast ratios meet 4.5:1 minimum for text on all 3 pages
- [ ] Assert status badges are accessible (text + color, not color alone)

## Acceptance Criteria

- [ ] axe-core reports zero violations on all 3 detail page types
- [ ] All action buttons have descriptive accessible names
- [ ] Form inputs and toggles are labeled
- [ ] Tables have proper scope attributes
- [ ] Keyboard navigation works on all detail pages
- [ ] Color contrast meets WCAG AA minimums
- [ ] Status badges convey meaning without color alone

## Definition of Done

- [ ] Playwright accessibility tests written and passing for all 3 detail page types
- [ ] Tests committed to `tests/playwright/detail-pages/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-74 tracker
