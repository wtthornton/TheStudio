# Story 74.1 — Page Intent & Semantic Content

**Epic:** 74 — Detail Pages (Repo/Workflow/Expert): Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Repo, Workflow, and Expert detail pages to clearly communicate their purpose through semantic HTML and meaningful content so that I can immediately understand the entity I am viewing and its key attributes.

## Description

Verify that the detail pages (`/admin/ui/repos/{repo_id}`, `/admin/ui/workflows/{workflow_id}`, `/admin/ui/experts/{expert_id}`) each render with correct semantic structure, convey the entity name/identifier as a heading, and display the expected domain content. Tests must seed or use existing test fixtures since detail pages require entity data.

### Repo Detail
Assert: tier, status, config, merge mode, promotion history vocabulary.

### Workflow Detail
Assert: pipeline stage, status, duration, action buttons vocabulary (Rerun, Escalate, Send-to-Agent).

### Expert Detail
Assert: trust tier, confidence, weight, drift analysis vocabulary.
