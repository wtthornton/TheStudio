# Story 74.6 — Visual Snapshot Baseline

**Epic:** 74 — Detail Pages (Repo/Workflow/Expert): Playwright Full-Stack Test Suite
**Points:** 2
**Status:** Draft

## User Story

As a platform operator, I want visual snapshot baselines for all detail pages so that any future UI regressions are caught automatically by comparing against the approved baselines.

## Description

Capture and commit Playwright visual snapshots for the Repo, Workflow, and Expert detail pages. Each detail page type should have its own set of baseline screenshots. Tests must seed or use existing test fixtures since detail pages require entity data. Snapshots are taken at a consistent viewport size and compared pixel-by-pixel in CI to catch unintended visual changes.

## Tasks

- [ ] Configure Playwright snapshot settings (viewport: 1280x720, threshold, maxDiffPixels)
- [ ] Capture baseline screenshot for Repo detail page (with seeded data)
- [ ] Capture baseline screenshot for Repo detail — merge mode section
- [ ] Capture baseline screenshot for Repo detail — promotion history section
- [ ] Capture baseline screenshot for Workflow detail page (with seeded data)
- [ ] Capture baseline screenshot for Workflow detail — pipeline stage visualization
- [ ] Capture baseline screenshot for Expert detail page (with seeded data)
- [ ] Capture baseline screenshot for Expert detail — drift analysis section
- [ ] Capture baseline screenshot for detail page 404/not-found state (invalid entity ID)
- [ ] Commit all baseline `.png` files to the snapshot directory
- [ ] Verify snapshot comparison passes on re-run (no false positives)

## Acceptance Criteria

- [ ] Baseline screenshots captured for all 3 detail page types and their key sections
- [ ] Snapshots taken at consistent viewport size (1280x720)
- [ ] Snapshot comparison passes on consecutive runs (deterministic)
- [ ] Baseline `.png` files committed to repository

## Definition of Done

- [ ] Playwright visual snapshot tests written and passing
- [ ] Baseline screenshots committed to `tests/playwright/detail-pages/__screenshots__/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-74 tracker
