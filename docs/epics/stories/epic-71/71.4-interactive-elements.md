# Story 71.4 — Interactive Elements

**Epic:** 71 — Settings: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want all interactive elements on the Settings page to function correctly so that I can switch between tabs, submit forms, reveal masked values, toggle feature flags, and trigger key rotation actions.

## Description

Verify that all interactive elements on the `/admin/ui/settings` page respond correctly to user input. This includes tab clicks loading content via HTMX swap, form submissions showing success/error feedback, the Reveal button unmasking API key values, toggle switches for feature flags, and Rotate/Regenerate buttons with confirmation behavior.

## Tasks

- [ ] Write Playwright test clicking each of the 6 tab links and asserting content swap via HTMX
- [ ] Assert tab click updates the active tab indicator
- [ ] Assert API Keys tab: Reveal button unmasks a key value (replaces *** with actual value)
- [ ] Assert API Keys tab: Create key form submits and shows success feedback
- [ ] Assert Infrastructure tab: form submits and shows success/error feedback
- [ ] Assert Feature Flags tab: toggle switch changes state (enabled/disabled) and persists
- [ ] Assert Agent Config tab: form submits with model/timeout/retries values
- [ ] Assert Budget Controls tab: form submits with budget limit values
- [ ] Assert Secrets tab: Rotate Key button triggers action (with confirmation if applicable)
- [ ] Assert Secrets tab: Regenerate Webhook button triggers action (with confirmation if applicable)
- [ ] Assert form validation prevents submission of invalid values where applicable
- [ ] Assert HTMX swap indicators (loading states) appear during tab transitions

## Acceptance Criteria

- [ ] All 6 tabs load content via HTMX when clicked
- [ ] Active tab indicator updates correctly
- [ ] Reveal button unmasks API key values
- [ ] Form submissions show success/error feedback
- [ ] Toggle switches change state and persist
- [ ] Destructive actions (rotate/regenerate) have confirmation or clear labeling
- [ ] No JavaScript errors in console during interactions

## Definition of Done

- [ ] Playwright interaction tests written and passing
- [ ] Test committed to `tests/playwright/settings/`
