# Story 71.1 — Page Intent & Semantic Content

**Epic:** 71 — Settings: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator, I want the Settings page to clearly communicate its purpose through semantic HTML and meaningful content so that I can immediately understand I am viewing the platform configuration hub with its 6 configuration tabs.

## Description

Verify that the `/admin/ui/settings` page renders with correct semantic structure, conveys the "Settings" heading, displays all 6 tab links (API Keys, Infrastructure, Feature Flags, Agent Config, Budget Controls, Secrets), and uses the expected domain vocabulary. Each tab should load its content via HTMX partial, and the default tab (API Keys) should render its content on initial load.

## Tasks

- [ ] Write Playwright test navigating to `/admin/ui/settings`
- [ ] Assert page contains an `<h1>` or prominent heading with "Settings"
- [ ] Assert semantic landmark structure (`<main>`, `<nav>`, `<header>` as applicable)
- [ ] Assert all 6 tab links are present (api-keys, infrastructure, feature-flags, agent-config, budget-controls, secrets)
- [ ] Assert default tab (API Keys) content renders on initial load
- [ ] Assert API Keys tab shows key names with masked values (***)
- [ ] Assert Infrastructure tab content includes connection strings, addresses, pool config vocabulary
- [ ] Assert Feature Flags tab shows toggleable flags (enabled/disabled)
- [ ] Assert Agent Config tab shows model, timeout, retries, concurrency vocabulary
- [ ] Assert Budget Controls tab shows budget limits and utilization vocabulary
- [ ] Assert Secrets tab shows rotation controls (rotate key, regenerate webhook)
- [ ] Validate no broken heading hierarchy (h1 > h2 > h3 in order)

## Acceptance Criteria

- [ ] Playwright test navigates to the settings page and asserts heading text
- [ ] Semantic landmarks are present and correct
- [ ] All 6 tab links are present and labeled
- [ ] Default tab content renders with expected domain vocabulary
- [ ] Each tab can be clicked and loads content via HTMX
- [ ] Heading hierarchy has no skipped levels

## Definition of Done

- [ ] Playwright test written and passing
- [ ] Test committed to `tests/playwright/settings/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-71 tracker
