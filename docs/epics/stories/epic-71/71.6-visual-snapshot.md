# Story 71.6 — Visual Snapshot Baseline

**Epic:** 71 — Settings: Playwright Full-Stack Test Suite
**Points:** 2
**Status:** Draft

## User Story

As a platform operator, I want visual snapshot baselines for the Settings page so that any future UI regressions are caught automatically by comparing against the approved baseline.

## Description

Capture and commit Playwright visual snapshots for the `/admin/ui/settings` page across all 6 tabs. Each tab should have its own baseline screenshot. Snapshots are taken at a consistent viewport size and compared pixel-by-pixel in CI to catch unintended visual changes.

## Tasks

- [ ] Configure Playwright snapshot settings (viewport: 1280x720, threshold, maxDiffPixels)
- [ ] Capture baseline screenshot for Settings page with default tab (API Keys)
- [ ] Capture baseline screenshot for Infrastructure tab
- [ ] Capture baseline screenshot for Feature Flags tab
- [ ] Capture baseline screenshot for Agent Config tab
- [ ] Capture baseline screenshot for Budget Controls tab
- [ ] Capture baseline screenshot for Secrets tab
- [ ] Capture baseline screenshot for empty state (if applicable)
- [ ] Commit all baseline `.png` files to the snapshot directory
- [ ] Verify snapshot comparison passes on re-run (no false positives)

## Acceptance Criteria

- [ ] Baseline screenshots captured for all 6 tabs plus the default view
- [ ] Snapshots taken at consistent viewport size (1280x720)
- [ ] Snapshot comparison passes on consecutive runs (deterministic)
- [ ] Baseline `.png` files committed to repository

## Definition of Done

- [ ] Playwright visual snapshot tests written and passing
- [ ] Baseline screenshots committed to `tests/playwright/settings/__screenshots__/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-71 tracker
