# Story 71.5 — Accessibility WCAG 2.2 AA

**Epic:** 71 — Settings: Playwright Full-Stack Test Suite
**Points:** 3
**Status:** Draft

## User Story

As a platform operator using assistive technology, I want the Settings page to be fully accessible so that I can navigate tabs, read configuration values, and interact with all controls using a screen reader or keyboard alone.

## Description

Verify that the `/admin/ui/settings` page meets WCAG 2.2 AA requirements. This includes axe-core scan with zero violations, correct ARIA patterns for the tab navigation (tablist/tab/tabpanel), labeled form inputs, accessible toggle switches, and keyboard operability for all interactive elements.

## Tasks

- [ ] Run axe-core accessibility scan on the settings page — assert zero violations
- [ ] Assert tab navigation uses `role="tablist"`, `role="tab"`, and `role="tabpanel"` ARIA pattern
- [ ] Assert `aria-selected="true"` on the active tab
- [ ] Assert `aria-controls` on tabs links to corresponding tabpanel `id`
- [ ] Assert all form inputs have associated `<label>` elements or `aria-label`
- [ ] Assert toggle switches have accessible names (`aria-label` or associated label)
- [ ] Assert toggle switches convey state (`aria-checked` or equivalent)
- [ ] Assert masked values are accessible (screen reader can identify masked vs revealed state)
- [ ] Assert action buttons (Reveal, Rotate, Regenerate) have descriptive accessible names
- [ ] Assert keyboard navigation: Tab key moves through interactive elements in logical order
- [ ] Assert keyboard navigation: Arrow keys navigate between tabs
- [ ] Assert keyboard navigation: Enter/Space activates buttons and toggles
- [ ] Assert focus indicators are visible on all focusable elements
- [ ] Assert color contrast ratios meet 4.5:1 minimum for text
- [ ] Run axe-core scan on each tab after switching — assert zero violations per tab

## Acceptance Criteria

- [ ] axe-core reports zero violations on initial load and after tab switches
- [ ] Tab navigation follows ARIA tablist/tab/tabpanel pattern
- [ ] All form inputs and controls are labeled
- [ ] Keyboard navigation works for all interactive elements
- [ ] Focus indicators are visible
- [ ] Color contrast meets WCAG AA minimums

## Definition of Done

- [ ] Playwright accessibility tests written and passing
- [ ] Test committed to `tests/playwright/settings/`
- [ ] No test flakiness on 3 consecutive runs
- [ ] Story checked off in epic-71 tracker
